📘 BLOC A — Fondamentaux d'électricité

A.1 — Les trois grandeurs

Tension U (volts, V) : différence de potentiel entre deux points. C'est ce que fournit le générateur (pile, alimentation, sortie 5V de l'Arduino). Analogie hydraulique : différence de hauteur entre deux bassins.

Courant I (ampères, A) : quantité de charges qui circule par seconde. Analogie : débit d'eau dans le tuyau. Pas de tension → pas de courant.

Résistance R (ohms, Ω) : ce qui s'oppose au passage du courant. Analogie : étranglement du tuyau.

Loi d'Ohm (à connaître par cœur)

U=R×IU = R \times IU=R×I

I=URI = \frac{U}{R}I=RU​

R=UIR = \frac{U}{I}R=IU​

Puissance P (watts, W)

P=U×IP = U \times IP=U×I

P=R×I2P = R \times I^2P=R×I2

P=U2RP = \frac{U^2}{R}P=RU2​

⚠️ Chaque résistance a une puissance max (typiquement 0,25 W ou 0,5 W). Si tu dépasses, elle brûle. À l'examen avec 5V, on est loin du danger.

A.3 — Série vs parallèle

En série (un seul chemin)

  • Même courant partout

  • La tension se répartit entre les composants

Reˊq=R1+R2+R3+…R_{éq} = R_1 + R_2 + R_3 + \ldotsReˊq​=R1​+R2​+R3​+…

En parallèle (plusieurs chemins)

  • Même tension sur chaque branche

  • Le courant se répartit entre les branches

1Reˊq=1R1+1R2+1R3+…\frac{1}{R_{éq}} = \frac{1}{R_1} + \frac{1}{R_2} + \frac{1}{R_3} + \ldotsReˊq​1​=R1​1​+R2​1​+R3​1​+…

Pour 2 résistances seulement (formule produit/somme) :

Reˊq=R1×R2R1+R2R_{éq} = \frac{R_1 \times R_2}{R_1 + R_2}Reˊq​=R1​+R2​R1​×R2​​

Lois de Kirchhoff

Loi des mailles : dans une boucle fermée, la somme des tensions vaut 0.

∑iUi=0\sum_i U_i = 0∑i​Ui​=0

Loi des nœuds : à un nœud, somme des entrants = somme des sortants.

∑Ientrants=∑Isortants\sum I_{entrants} = \sum I_{sortants}∑Ientrants​=∑Isortants​

Pont diviseur de tension (utile partout)

UR2=Usource×R2R1+R2U_{R_2} = U_{source} \times \frac{R_2}{R_1 + R_2}UR2​​=Usource​×R1​+R2​R2​​

A.4 — La LED (Light Emitting Diode)

Composant polarisé : elle ne fonctionne que dans un seul sens.

Caractéristiques typiques

  • Tension directe V_F ≈ 2 V (rouge/verte) ou ~3 V (bleue/blanche)

  • Courant nominal I_F ≈ 20 mA = 0,02 A

⚠️ JAMAIS de LED branchée seule sur 5V → courant énorme → la LED claque instantanément. Toujours une résistance en série.

Calcul de la résistance de limitation

R=Vsource−VFIFR = \frac{V_{source} - V_F}{I_F}R=IF​Vsource​−VF​​

Application numérique pour une LED rouge sur 5V :

R=5−20,02=150 ΩR = \frac{5 - 2}{0{,}02} = 150 \ \OmegaR=0,025−2​=150 Ω

En pratique : 220 Ω ou 330 Ω (marge de sécurité, LED un peu moins lumineuse mais ça vit).

📘 BLOC B — Composants spéciaux

B.1 — La photorésistance (LDR)

LDR = Light Dependent Resistor. Sa résistance varie avec la lumière.

Comportement (contre-intuitif !)

  • Beaucoup de lumière → résistance faible (~1 kΩ)

  • Obscurité → résistance élevée (~100 kΩ à 1 MΩ)

Montage en pont diviseur

L'Arduino ne mesure pas de résistance, seulement des tensions. On utilise donc un pont diviseur avec une résistance fixe (typiquement 10 kΩ).

   +5V (Arduino)
     │
    ┌┴┐
    │R│  10 kΩ (résistance fixe)
    └┬┘
     ●──── A0 (entrée analogique)
    ┌┴┐
    │LDR│
    └┬┘
     │
    GND

Formule du pont diviseur

UA0=5×RLDRRfixe+RLDRU_{A0} = 5 \times \frac{R_{LDR}}{R_{fixe} + R_{LDR}}UA0​=5×Rfixe​+RLDR​RLDR​​

Pourquoi 10 kΩ ?

C'est l'ordre de grandeur médian entre la LDR en pleine lumière (~1 kΩ) et dans le noir (~100 kΩ). Ça maximise la plage de variation de la tension lue en A0.

  • En pleine lumière : U_A0 ≈ 0,45 V

  • Dans le noir : U_A0 ≈ 4,5 V

Lecture côté Arduino

int valeur = analogRead(A0);            // 0 à 1023
float tension = valeur * 5.0 / 1023.0;  // conversion en volts

Pourquoi une entrée analogique ?

Parce que la LDR varie continuellement avec la lumière. Une entrée numérique ne lit que HIGH/LOW (au-dessus ou en dessous de ~2,5V), ce qui perdrait toute l'information. L'entrée analogique mesure une valeur proportionnelle à la tension, avec 10 bits de résolution (1024 niveaux).

B.2 — Le transistor NPN en commutation

Interrupteur électronique commandé : un petit courant ouvre/ferme le passage d'un gros courant.

Trois broches

Broche

Rôle

Collecteur (C)

Où entre le courant principal (côté +)

Base (B)

La commande (le petit courant qui pilote)

Émetteur (E)

Où sort le courant (côté GND)

⚠️ Le brochage dépend du modèle ! Toujours vérifier la datasheet (boîtier 2N2222, BC547, etc.).

Les deux modes utiles

  • Bloqué (I_B ≈ 0) → aucun courant C→E → charge éteinte

  • Saturé (un faible I_B suffit) → gros courant C→E → charge allumée

Formules à connaître

IC=β×IBI_C = \beta \times I_BIC​=β×IB​

IE=IC+IBI_E = I_C + I_BIE​=IC​+IB​

VBE≈0,7 VV_{BE} \approx 0{,}7 \text{ V}VBE​≈0,7 V

β (bêta ou h_FE) = gain en courant, typiquement 100-200.

Méthode pour vérifier la saturation

Étape 1 — Courant de collecteur en saturation :

IC,sat=VCC−VLEDRCI_{C,sat} = \frac{V_{CC} - V_{LED}}{R_C}IC,sat​=RC​VCC​−VLED​​

Étape 2 — Courant de base minimum pour saturer :

IB,min=IC,satβI_{B,min} = \frac{I_{C,sat}}{\beta}IB,min​=βIC,sat​​

Étape 3 — Courant de base réel fourni :

IB=Vcommande−0,7RBI_B = \frac{V_{commande} - 0{,}7}{R_B}IB​=RB​Vcommande​−0,7​

Étape 4 — Conclusion :

Si I_B ≥ I_B,min → transistor saturé ✅ Sinon → la LED n'est pas pleinement allumée.

Exemple complet (labo 5, β = 100)

LED V_F = 2V, R_C = 470Ω, R_B = 10kΩ, commande 5V.

Calcul de I_C,sat :

IC,sat=5−2470=6,4 mAI_{C,sat} = \frac{5 - 2}{470} = 6{,}4 \text{ mA}IC,sat​=4705−2​=6,4 mA

Calcul de I_B,min :

IB,min=6,4 mA100=64 μAI_{B,min} = \frac{6{,}4 \text{ mA}}{100} = 64 \ \mu\text{A}IB,min​=1006,4 mA​=64 μA

Calcul de I_B réel :

IB=5−0,710,000=430 μAI_B = \frac{5 - 0{,}7}{10,000} = 430 \ \mu\text{A}IB​=10,0005−0,7​=430 μA

Conclusion : 430 µA ≫ 64 µA → largement saturé ✅

Variante avec R_B = 100 kΩ :

IB=5−0,7100,000=43 μAI_B = \frac{5 - 0{,}7}{100,000} = 43 \ \mu\text{A}IB​=100,0005−0,7​=43 μA

43 µA < 64 µA → pas saturé.

Pourquoi utiliser un transistor ?

Une broche Arduino fournit max 40 mA (20 mA recommandé). Pour piloter un moteur, un relais ou plusieurs LEDs en parallèle, c'est insuffisant. Le transistor amplifie : la broche fournit un petit I_B, et le gros courant passe par l'alim 5V via le collecteur — sans surcharger l'Arduino.

C.5 — Anatomie de l'Arduino UNO

Power

  • 5V : sortie 5V

  • 3.3V : sortie 3,3V

  • GND : masse (plusieurs broches)

  • VIN : entrée alim externe 7-12V

  • Reset : redémarre le programme

Broches numériques (D0 à D13)

  • D0 (RX), D1 (TX) : ⚠️ Communication série — ne pas utiliser pour BP/LED

  • D2 à D13 : I/O numériques standard

  • D3, D5, D6, D9, D10, D11 : marquées ~ → broches PWM (analogWrite)

  • D13 : LED intégrée sur la carte

Broches analogiques (A0 à A5)

  • Convertisseur 10 bits → analogRead retourne 0 à 1023

  • Plage 0V à 5V → résolution ≈ 4,9 mV/unité

Limites électriques

  • 40 mA max par broche (20 mA recommandé)

  • 200 mA total toutes broches confondues

  • 0V à 5V uniquement (jamais de tension négative ou > 5V)

D.3 — Pull-up (LE piège classique)

Le problème

Sans résistance, une broche en entrée flotte → lecture aléatoire.

Solution : INPUT_PULLUP

pinMode(2, INPUT_PULLUP);

Câblage : bouton entre broche et GND. Une résistance interne (~50kΩ) tire la broche vers 5V.

⚠️ Logique inversée

  • Bouton relâché → HIGH

  • Bouton appuyé → LOW (connecté à GND)

if (digitalRead(2) == LOW) {
    // bouton appuyé
}

D.4 — millis() vs delay()

delay(1000) = "attends 1s, ne fais rien d'autre". Bloque tout le programme.

millis() = nombre de ms écoulées depuis le démarrage. Permet de vérifier le temps sans bloquer.

Pattern non-bloquant

unsigned long tempsPrecedent = 0;
const unsigned long INTERVALLE = 500;
bool etatLED = LOW;

void loop() {
    unsigned long maintenant = millis();

    if (maintenant - tempsPrecedent >= INTERVALLE) {
        tempsPrecedent = maintenant;
        etatLED = !etatLED;
        digitalWrite(LED, etatLED);
    }
    // le reste du loop continue librement
}

⚠️ Toujours unsigned long pour les variables de temps (un int déborde après ~32 s).

D.5 — Détection de front + anti-rebond

Détection sur front (pas sur état)

bool etatBP_precedent = HIGH;

void loop() {
    bool etatBP_actuel = digitalRead(BP);

    // Front descendant : passage HIGH → LOW
    if (etatBP_precedent == HIGH && etatBP_actuel == LOW) {
        // action déclenchée UNE SEULE FOIS
    }

    etatBP_precedent = etatBP_actuel;  // mémoriser pour la suite
}

Anti-rebond avec millis()

const unsigned long ANTI_REBOND = 50;

if (etatBP_precedent == HIGH && etatBP_actuel == LOW) {
    if (millis() - tempsDernierAppui > ANTI_REBOND) {
        // action
        tempsDernierAppui = millis();
    }
}

Un bouton mécanique génère des micro-impulsions parasites pendant ~10-50 ms après l'appui. Sans anti-rebond, un appui est détecté 3-5 fois.

D.6 — Appui court vs appui long

unsigned long tempsDebutAppui = 0;
bool bp_etaitAppuye = false;
const unsigned long SEUIL_LONG = 2000;  // 2s

void loop() {
    bool bp_appuye = (digitalRead(BP) == LOW);

    // Front descendant
    if (bp_appuye && !bp_etaitAppuye) {
        tempsDebutAppui = millis();
        bp_etaitAppuye = true;
    }

    // Front montant : on calcule la durée
    if (!bp_appuye && bp_etaitAppuye) {
        unsigned long duree = millis() - tempsDebutAppui;
        bp_etaitAppuye = false;

        if (duree < SEUIL_LONG) {
            // appui court → télérupteur
        } else {
            // appui long → autre action (clignotement)
        }
    }
}

⚡ Q2 — Télérupteur avec bouton poussoir

Énoncé

À chaque appui simple, la LED change d'état et reste dans cet état jusqu'au prochain appui.

Câblage

D13 ─── 220Ω ─── anode LED ─── cathode ─── GND
D2  ─── bouton  ─── GND       (INPUT_PULLUP, pas de R externe)

Code

#define LED 13
#define BP  2

bool etatLED = LOW;
bool etatBP_precedent = HIGH;
unsigned long tempsDernierAppui = 0;
const unsigned long ANTI_REBOND = 50;

void setup() {
    pinMode(LED, OUTPUT);
    pinMode(BP, INPUT_PULLUP);
    digitalWrite(LED, etatLED);
}

void loop() {
    bool etatBP_actuel = digitalRead(BP);

    // Front descendant
    if (etatBP_precedent == HIGH && etatBP_actuel == LOW) {
        if (millis() - tempsDernierAppui > ANTI_REBOND) {
            etatLED = !etatLED;
            digitalWrite(LED, etatLED);
            tempsDernierAppui = millis();
        }
    }

    etatBP_precedent = etatBP_actuel;
}

Questions probables du prof

  • Pourquoi INPUT_PULLUP ? → évite entrée flottante (lecture aléatoire)

  • Pourquoi logique inversée ? → pull-up tire vers 5V, appui = court-circuit vers GND

  • C'est quoi un rebond ? → micro-impulsions parasites du bouton mécanique (10-50 ms)

  • Pourquoi pas delay() ? → delay() bloque tout le programme

⚡ Q3 — Deuxième LED + appui long = clignotement 0,6 Hz

Énoncé

Ajouter une seconde LED. Appui court → télérupteur LED1. Appui long (>2s) → les 2 LEDs clignotent à 0,6 Hz.

Calcul de la demi-période

T=10,6≈1666 msT = \frac{1}{0{,}6} \approx 1666 \text{ ms}T=0,61​≈1666 ms

demi-peˊriode=T2=833 ms\text{demi-période} = \frac{T}{2} = 833 \text{ ms}demi-peˊriode=2T​=833 ms

Câblage

D13 ─── 220Ω ─── LED1 ─── GND
D12 ─── 220Ω ─── LED2 ─── GND     (couleur différente)
D2  ─── bouton ─── GND

Code

#define LED1 13
#define LED2 12
#define BP   2

bool etatLED1 = LOW;
bool etatLED2 = LOW;
bool modeClignotement = false;

bool bp_etaitAppuye = false;
unsigned long tempsDebutAppui = 0;
const unsigned long ANTI_REBOND = 50;
const unsigned long SEUIL_LONG  = 2000;

const unsigned long DEMI_PERIODE = 833;
unsigned long tempsDerniereBascule = 0;

void setup() {
    pinMode(LED1, OUTPUT);
    pinMode(LED2, OUTPUT);
    pinMode(BP, INPUT_PULLUP);
    digitalWrite(LED1, etatLED1);
    digitalWrite(LED2, etatLED2);
}

void loop() {
    // ── Partie 1 : bouton ──
    bool bp_appuye = (digitalRead(BP) == LOW);

    if (bp_appuye && !bp_etaitAppuye) {
        if (millis() - tempsDebutAppui > ANTI_REBOND) {
            tempsDebutAppui = millis();
            bp_etaitAppuye = true;
        }
    }

    if (!bp_appuye && bp_etaitAppuye) {
        unsigned long duree = millis() - tempsDebutAppui;
        bp_etaitAppuye = false;

        if (duree < SEUIL_LONG) {
            // Appui court → télérupteur LED1
            modeClignotement = false;
            digitalWrite(LED2, LOW);
            etatLED2 = LOW;
            etatLED1 = !etatLED1;
            digitalWrite(LED1, etatLED1);
        } else {
            // Appui long → clignotement
            modeClignotement = true;
            tempsDerniereBascule = millis();
        }
    }

    // ── Partie 2 : clignotement ──
    if (modeClignotement) {
        unsigned long maintenant = millis();
        if (maintenant - tempsDerniereBascule >= DEMI_PERIODE) {
            tempsDerniereBascule = maintenant;
            etatLED1 = !etatLED1;
            etatLED2 = !etatLED2;
            digitalWrite(LED1, etatLED1);
            digitalWrite(LED2, etatLED2);
        }
    }
}

Questions probables du prof

  • Calcul de 833 ms ? → T = 1/0,6 ≈ 1666 ms, demi-période = 833 ms

  • Pourquoi 2 parties dans le loop ? → tâches indépendantes non-bloquantes

  • Que se passe-t-il si on fait un appui court pendant le clignotement ? → arrêt clignotement, retour télérupteur LED1

  • Pourquoi unsigned long ? → millis() retourne unsigned long ; int déborderait en ~32s

bool etatLED1 = LOW;
bool etatLED2 = LOW;
bool modeClignotement = false;

bool bp_etaitAppuye = false;
unsigned long tempsDebutAppui = 0;
const unsigned long ANTI_REBOND = 50;
const unsigned long SEUIL_LONG  = 2000;

const unsigned long DEMI_PERIODE = 833;
unsigned long tempsDerniereBascule   = 0;
unsigned long tempsBasculePrecedente = 0;

void setup() {
    pinMode(LED1, OUTPUT);
    pinMode(LED2, OUTPUT);
    pinMode(BP, INPUT_PULLUP);
    digitalWrite(LED1, etatLED1);
    digitalWrite(LED2, etatLED2);
    Serial.begin(9600);
}

void loop() {
    // ── Partie 1 : bouton ──
    bool bp_appuye = (digitalRead(BP) == LOW);

    if (bp_appuye && !bp_etaitAppuye) {
        if (millis() - tempsDebutAppui > ANTI_REBOND) {
            tempsDebutAppui = millis();
            bp_etaitAppuye = true;
        }
    }

    if (!bp_appuye && bp_etaitAppuye) {
        unsigned long duree = millis() - tempsDebutAppui;
        bp_etaitAppuye = false;

        if (duree < SEUIL_LONG) {
            Serial.print("Appui court - duree : ");
            Serial.print(duree);
            Serial.println(" ms");

            modeClignotement = false;
            digitalWrite(LED2, LOW);
            etatLED2 = LOW;
            etatLED1 = !etatLED1;
            digitalWrite(LED1, etatLED1);
        } else {
            Serial.print("Appui long - duree : ");
            Serial.print(duree);
            Serial.println(" ms");

            modeClignotement = true;
            tempsDerniereBascule   = millis();
            tempsBasculePrecedente = millis();
        }
    }

    // ── Partie 2 : clignotement + mesure période ──
    if (modeClignotement) {
        unsigned long maintenant = millis();

        if (maintenant - tempsDerniereBascule >= DEMI_PERIODE) {
            unsigned long demiPeriodeMesuree = maintenant - tempsBasculePrecedente;
            float periodeMesuree = (demiPeriodeMesuree * 2) / 1000.0;

            Serial.print("Periode mesuree : ");
            Serial.print(periodeMesuree, 3);
            Serial.println(" s");

            etatLED1 = !etatLED1;
            etatLED2 = !etatLED2;
            digitalWrite(LED1, etatLED1);
            digitalWrite(LED2, etatLED2);

            tempsBasculePrecedente = tempsDerniereBascule;
            tempsDerniereBascule   = maintenant;
        }
    }
}

Questions probables du prof

  • Pourquoi Serial.begin() dans setup ? → init une seule fois ; sinon réinit chaque loop

  • Pourquoi /1000.0 et pas /1000 ? → forcer division en float, sinon décimales perdues

  • Caractères bizarres dans le moniteur ? → baud rate différent entre code et moniteur

  • Pourquoi mesurer la période réelle ? → variations dues au temps d'exécution

⚠️ Pièges classiques à éviter

  1. Oublier la résistance sur la LED → claquage → –6 pts

  2. Utiliser int pour millis() → débordement en ~32 s → bug imprévisible

  3. delay() interdit dans les cahiers des charges → toujours millis()

  4. Confondre analogRead (0–1023) et analogWrite (0–255) → utiliser map()

  5. Pull-up oublié → bouton imprévisible → utiliser INPUT_PULLUP

  6. Logique inversée avec INPUT_PULLUP : LOW = appuyé, HIGH = relâché

  7. Anti-rebond oublié → télérupteur déclenche 2-3 fois

  8. Détection sur état au lieu de front → action répétée tant que BP maintenu

  9. Serial.begin() oublié → moniteur série ne montre rien

  10. Broches 0 et 1 utilisées → upload bloqué (RX/TX réservés)

  11. Mauvais baud rate → caractères incompréhensibles

  12. Ampèremètre en parallèle → court-circuit

  13. Variables non remises à zéro entre les états → comportement incohérent
