📡 Labo 6 — Communication Série Arduino

🎯 Objectif général

Apprendre à faire communiquer l'Arduino avec le PC via la liaison série (UART sur USB). Cela permet d'afficher des valeurs dans le Moniteur Série de l'IDE Arduino (pour debug ou affichage de capteurs) et de recevoir des commandes depuis le PC.

🛠️ Matériel & outils

Logiciel

  • Arduino IDE → tout ce qu'il faut (écrire le code, téléverser, ouvrir le Moniteur Série)

  • Wokwi / VS Code → optionnel, simulateur ou alternative

Matériel

  • Carte Arduino Uno (ou Elegoo Uno R3, 100% compatible)

  • Câble USB

  • Breadboard

  • 1 bouton poussoir

  • 1 LED

  • 1 photorésistance (LDR)

  • Résistances : 10 kΩ (pull-down et pont diviseur), 330 Ω (protection LED)

  • Câbles mâle-mâle

⚙️ Configuration de l'IDE Arduino (à faire 1 fois)

  1. Brancher l'Arduino en USB

  2. Outils → Type de carte → Arduino Uno

  3. Outils → Port → sélectionner le port COM correspondant

  4. Tester avec Fichier → Exemples → 01.Basics → Blink : si la LED de la carte clignote → tout fonctionne ✅

💡 Astuce port COM : débranche l'Arduino, regarde les ports listés, rebranche-le → le nouveau port qui apparaît est le bon.

📚 Théorie — La communication série

L'Arduino n'a pas d'écran → pour afficher des messages, il les envoie au PC via UART (couche liaison) sur USB (couche physique).

Pour que la communication fonctionne, les deux côtés doivent utiliser la même vitesse de transmission (baud rate), typiquement 9600.

Fonctions principales

Serial.begin(9600);          // Initialise la connexion (dans setup())

// Envoyer du PC vers le moniteur
Serial.print("texte");       // Affiche sans saut de ligne
Serial.println("texte");     // Affiche avec saut de ligne

// Recevoir des messages depuis le PC
Serial.available();          // Renvoie le nombre d'octets dans le buffer
Serial.readString();         // Lit tout le buffer comme une String
Serial.read();               // Lit 1 byte (1 caractère)

⚠️ Pièges à connaître

  • Pin 0 (RX) et Pin 1 (TX) : utilisées par la communication série → ne pas les utiliser pour autre chose, sinon le téléversement et les print peuvent planter.

  • Moniteur Série — Fin de ligne : en bas du moniteur, il y a un menu déroulant. Pour comparer des messages avec ==, il faut être sur "Pas de fin de ligne", sinon des caractères invisibles (\n ou \r) s'ajoutent au message et la comparaison échoue.

  • Baud rate : doit être identique dans le code (Serial.begin(9600)) et dans le moniteur série (menu en bas).

📦 Le buffer série — comment ça marche ?

Quand le PC envoie un message à l'Arduino, les octets arrivent et s'accumulent dans une zone mémoire temporaire appelée le buffer (file d'attente). L'Arduino peut ensuite lire ce buffer quand il est prêt.

Serial.available() permet de vérifier si des octets attendent dans le buffer avant d'essayer de les lire.

Scénario 1 — Sans Serial.available(), buffer vide :

  • Serial.readString() est appelé alors qu'il n'y a rien

  • L'Arduino attend un timeout (~1 seconde) avant de continuer

  • Tout le reste du loop() est bloqué pendant ce temps

Scénario 2 — Avec Serial.available(), buffer vide :

  • Serial.available() retourne 0 → le if est faux → on passe directement à la suite

  • Rien n'est bloqué, le programme continue normalement ✅

Scénario 3 — Avec Serial.available(), message reçu ("ON") :

  • Les octets 'O' et 'N' sont dans le buffer

  • Serial.available() retourne 2 → le if est vrai → on entre dedans

  • Serial.readString() lit tout le buffer et retourne "ON" ✅

  • Le buffer est ensuite vide, prêt pour le prochain message

💡 C'est comme vérifier ta boîte aux lettres avant de l'ouvrir : inutile de l'ouvrir si elle est vide !

Outils du moniteur

  • Moniteur Série (loupe 🔍 en haut à droite) → affiche les messages texte.

  • Traceur Série (Outils → Traceur Série) → affiche un graphique d'une valeur entière passée à Serial.println(). Très pratique pour visualiser un capteur.

🧪 Les exercices

Exercice 1 — Compter les secondes d'appui sur un bouton

Objectif : Quand on appuie sur un bouton, compter combien de secondes il reste enfoncé et afficher la durée.

🔌 Circuit

Montage avec résistance pull-down :

       5V ──────┐
                │
              [BTN]
                │
   pin 2 ───────┤
                │
              [10kΩ]
                │
              GND

  • Une patte du bouton → 5V

  • L'autre patte du bouton → pin 2 ET résistance 10 kΩ vers GND

💡 La résistance de pull-down force la pin 2 à lire 0 (LOW) quand le bouton est relâché. Sans elle, la pin 2 serait "flottante" et lirait n'importe quoi.

💻 Code

#define BP 2  // Bouton poussoir sur la pin 2

void setup() {
    pinMode(BP, INPUT);    // Bouton en entrée
    Serial.begin(9600);    // Initialise la liaison série à 9600 bauds
}

void loop() {
    // Si le bouton est appuyé
    if (digitalRead(BP) == HIGH) {
        long debut = millis();   // On mémorise le temps de départ (en ms)
        
        // Tant que le bouton reste enfoncé, on attend
        while (digitalRead(BP) == HIGH) {
            // boucle vide : on attend simplement le relâchement
        }
        
        // Calcul de la durée
        long duree = millis() - debut;
        Serial.print("Durée d'appui : ");
        Serial.print(duree / 1000.0);   // Conversion ms → secondes
        Serial.println(" secondes");
    }
}

💡 Notions clés

  • millis() → renvoie le nombre de millisecondes écoulées depuis le démarrage de l'Arduino.

  • Pour mesurer une durée : durée = millis_fin - millis_début.

  • duree / 1000.0 → la division par 1000.0 (avec le point) force le calcul en flottant pour avoir les décimales.

Exercice 2 — Afficher l'alphabet avec un bouton

Objectif : Quand on appuie sur le bouton, afficher les 26 lettres de l'alphabet (200 ms entre chaque). Si le bouton reste appuyé après Z, recommencer depuis A.

🔌 Circuit

Même circuit que l'exercice 1 (bouton sur pin 2 avec pull-down).

💻 Code

#define BP 2

void setup() {
    pinMode(BP, INPUT);
    Serial.begin(9600);
}

void loop() {
    // Si le bouton est appuyé, on affiche l'alphabet
    if (digitalRead(BP) == HIGH) {
        // Boucle sur l'alphabet en utilisant l'ASCII
        for (char lettre = 'A'; lettre <= 'Z'; lettre++) {
            Serial.println(lettre);
            delay(200);   // 200 ms entre chaque lettre
        }
        // Après Z, loop() recommence → si le bouton est toujours appuyé, ça repart depuis A
    }
}

💡 Notions clés

  • En C++, les caractères sont des nombres ASCII :

    • 'A' = 65, 'B' = 66, ..., 'Z' = 90

  • lettre++ incrémente la valeur ASCII → on passe à la lettre suivante.

  • Pas besoin de while ici : après Z, loop() repart depuis le début et le if re-vérifie le bouton → si toujours appuyé, l'alphabet recommence.

⚠️ Comportement à savoir

Un appui rapide (< 5.2 s = 26 × 200 ms) lance la boucle for et la laisse finir jusqu'à Z avant de s'arrêter. La boucle ne vérifie pas le bouton à chaque lettre. C'est un comportement attendu pour ce labo.

Exercice 3 — Lire une photorésistance en continu

Objectif : Afficher en continu la valeur de luminosité captée par une photorésistance.

🔌 Circuit — Pont diviseur de tension

       5V ─────┐
                │
              [LDR]   (photorésistance)
                │
   A0  ─────────┤
                │
              [10kΩ]
                │
              GND

  • Photorésistance entre 5V et la pin A0

  • Résistance 10 kΩ entre A0 et GND

💻 Code

void setup() {
    Serial.begin(9600);
    // Pas besoin de pinMode pour les entrées analogiques
}

void loop() {
    int luminosite = analogRead(A0);   // Lecture entre 0 et 1023
    Serial.println(luminosite);
    delay(200);                        // 200 ms entre chaque lecture
}

💡 Notions clés

  • analogRead(A0) → renvoie une valeur entre 0 (0V) et 1023 (5V).

  • Plus il y a de lumière → moins de résistance dans la LDR → tension plus élevée sur A0 → valeur plus grande.

  • Pour visualiser graphiquement : Outils → Traceur Série 📈

❓ Pourquoi une entrée analogique ?

Une LDR a une résistance qui varie continûment avec la lumière. Une entrée digitale ne donnerait que 0 ou 1. Une entrée analogique permet de mesurer toute la gamme intermédiaire.

Exercice 4 — Contrôler une LED depuis le Moniteur Série

Objectif : Envoyer ON ou OFF depuis le Moniteur Série pour allumer ou éteindre une LED.

🔌 Circuit

   pin 8 ──[330Ω]──┤▶├── GND
                  LED

  • Longue patte de la LED (anode +) → résistance 330 Ω → pin 8

  • Courte patte de la LED (cathode -) → GND

💡 La résistance 330 Ω protège la LED. Elle peut être avant ou après la LED, ça ne change rien.

💻 Code

#define LED 8

void setup() {
    pinMode(LED, OUTPUT);
    Serial.begin(9600);
}

void loop() {
    // Si un message est arrivé du PC
    if (Serial.available()) {
        String message = Serial.readString();
        
        if (message == "ON") {
            digitalWrite(LED, HIGH);
            Serial.println("LED allumée !");
        } 
        else if (message == "OFF") {
            digitalWrite(LED, LOW);
            Serial.println("LED éteinte !");
        }
        else {
            Serial.println("Message inconnu, envoie ON ou OFF");
        }
    }
}

⚠️ ⚠️ TRÈS IMPORTANT — Fin de ligne ⚠️ ⚠️

Dans le Moniteur Série, en bas, il y a un menu déroulant à côté du baud rate.

→ Mettre sur "Pas de fin de ligne"

Sinon, le moniteur envoie ON\n au lieu de ON, et la comparaison message == "ON" échoue car les caractères invisibles ne correspondent pas.

💡 Notions clés

  • Serial.available() → renvoie le nombre d'octets reçus. Si > 0 → un message attend dans le buffer.

  • Serial.readString() → lit tout le buffer et le renvoie comme String.

  • Comparaison == de String fonctionne directement en Arduino.

Exercice 5 — Luminosité sur demande

Objectif : Comme l'exercice 3, mais la valeur n'est affichée que lorsqu'on envoie Lumi dans le Moniteur Série.

🔌 Circuit

Photorésistance sur A0 (comme l'exercice 3). La LED de l'exercice 4 peut rester branchée, elle ne gêne pas.

💻 Code

void setup() {
    Serial.begin(9600);
}

void loop() {
    // Si un message est arrivé du PC
    if (Serial.available()) {
        String message = Serial.readString();
        
        // Si on reçoit "Lumi" → on lit et on affiche la luminosité
        if (message == "Lumi") {
            int luminosite = analogRead(A0);
            Serial.print("Luminosité : ");
            Serial.println(luminosite);
        }
        else {
            Serial.println("Message inconnu, envoie Lumi");
        }
    }
}

💡 Notions clés

  • C'est la combinaison des exercices 3 et 4.

  • Mode "requête/réponse" : l'Arduino n'envoie une donnée que sur demande, ce qui économise la bande passante et évite de spammer le moniteur.

  • Toujours vérifier le "Pas de fin de ligne" dans le moniteur !

✅ Checklist avant de coder

  • [ ] Arduino branché en USB

  • [ ] Bon type de carte sélectionné (Outils → Type de carte → Uno)

  • [ ] Bon port COM sélectionné (Outils → Port)

  • [ ] Moniteur Série ouvert sur 9600 bauds

  • [ ] Menu fin de ligne sur "Pas de fin de ligne" (pour les ex. 4 et 5)

  • [ ] Circuit monté et vérifié (notamment les pull-down pour les boutons)

🐛 Pièges classiques à éviter

  1. Bouton sans pull-down/pull-up → la pin lit n'importe quoi → utiliser une résistance 10 kΩ.

  2. Moniteur Série fermé → on ne voit rien s'afficher → ouvrir avec la loupe 🔍.

  3. Mauvais baud rate → caractères illisibles → mettre 9600 partout.

  4. "Nouvelle ligne" activé dans le moniteur → les comparaisons == échouent → mettre "Pas de fin de ligne".

  5. Polarité de la LED inversée → ne s'allume pas → longue patte = +, courte patte = -.

  6. Utiliser les pins 0 ou 1 → bloque la liaison série et le téléversement → ne pas les utiliser.
