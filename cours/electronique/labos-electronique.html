⚠️ Règles de sécurité — ne pas perdre de points bêtement

  1. JAMAIS de LED sans résistance → claquage instantané (–6 pts à l'examen).

  2. Sens de la LED : anode (longue patte) côté +, cathode (courte patte / méplat) côté –/GND.

  3. Sens du transistor NPN : repère bien Collecteur / Base / Émetteur (datasheet).

  4. Mesure de courant : ouvrir le circuit, ampèremètre en série, sinon court-circuit.

  5. Mesure de tension : voltmètre en parallèle, jamais en série.

  6. Avant de brancher : alimentation éteinte ou Arduino débranché.

  7. Pull-up / pull-down obligatoire sur un BP, sinon entrée flottante.

🧭 Méthodologie générale

  1. Lire l'énoncé en entier avant de toucher au matériel.

  2. Dessiner le schéma (papier ou tête) avant le câblage.

  4. Câbler sur breadboard, vérifier deux fois.

  6. Coder par étapes, tester chaque étape avant la suivante.

1.1 Matériel & mesures

L'alimentation de laboratoire

  • Bornes orange : tension fixe 5V et 12V DC.

  • Bornes jaunes : tension ajustable.

  • Réglage rouge : tension de sortie (0 à 30V).

  • Réglage bleu : courant maximal (limite de sécurité).

  • Pour régler ImaxI_{max}Imax​ : court-circuiter la sortie, l'alim passe en mode « Current Limited ».

Le multimètre

Tension (V)

  • Branchement : en parallèle avec le composant mesuré

  • Résistance interne du voltmètre : quasi infinie (le courant ne passe pas dedans)

  • Connecteur : V/Ω

Courant (A)

  • Branchement : en série — il faut ouvrir le circuit pour l'insérer

  • Résistance interne de l'ampèremètre : quasi nulle

  • Connecteur : 10A ou 400mA selon le courant attendu

Résistance (Ω)

  • Mesure circuit HORS tension uniquement

  • Connecteur : V/Ω

⚠️ Erreur fatale : ampèremètre en parallèle = court-circuit garanti.

L'oscilloscope (labo 4)

  • Permet de mesurer une tension variable dans le temps.

  • Sert pour visualiser : signal alternatif, ondulation de filtrage, PWM, signal redressé.

  • Mesures utiles : amplitude crête à crête (VppV_{pp}Vpp​), période (TTT), fréquence (f=1/Tf = 1/Tf=1/T).

La breadboard

  • Lignes horizontales (au milieu) : connectées par groupes de 5 trous, séparées par la rainure centrale.

  • Lignes verticales (les 2 grandes bandes latérales) : utilisées pour le + et le –, coupées au milieu de la plaque.

1.2 Circuits série / parallèle / mixte (labos 1, 2, 3)

Lois fondamentales (à connaître par cœur)

U=R×IP=U×IU = R \times I \qquad P = U \times IU=R×IP=U×I

Kirchhoff — loi des mailles : ∑iUi=0\displaystyle\sum_i U_i = 0i∑​Ui​=0 dans une boucle fermée.

Kirchhoff — loi des nœuds : ∑Ientrants=∑Isortants\displaystyle\sum I_{entrants} = \sum I_{sortants}∑Ientrants​=∑Isortants​

Résistances équivalentes

En série : Reˊq=R1+R2+R3+…R_{éq} = R_1 + R_2 + R_3 + \ldotsReˊq​=R1​+R2​+R3​+…

En parallèle (formule générale) : 1Reˊq=1R1+1R2+1R3+…\frac{1}{R_{éq}} = \frac{1}{R_1} + \frac{1}{R_2} + \frac{1}{R_3} + \ldotsReˊq​1​=R1​1​+R2​1​+R3​1​+…

En parallèle (2 résistances uniquement) : Reˊq=R1×R2R1+R2R_{éq} = \frac{R_1 \times R_2}{R_1 + R_2}Reˊq​=R1​+R2​R1​×R2​​

Comportement à retenir

En série :

  • Le courant est identique partout dans la boucle

  • La tension se divise entre les composants

  • La résistance équivalente est plus grande que chaque résistance seule

  • Loi de Kirchhoff applicable : loi des mailles

En parallèle :

  • La tension est identique sur chaque branche

  • Le courant se divise entre les branches

  • La résistance équivalente est plus petite que la plus petite résistance

  • Loi de Kirchhoff applicable : loi des nœuds

Manipulation type

  1. Mesurer chaque résistance au multimètre — vérifier la valeur attendue.

  2. Réaliser le circuit sur LTSpice (simulation) puis sur breadboard (réel).

  3. Mesurer UR1U_{R1}UR1​, UR2U_{R2}UR2​, UR3U_{R3}UR3​, UtotU_{tot}Utot​, puis IR1I_{R1}IR1​, IR2I_{R2}IR2​, IR3I_{R3}IR3​, ItotI_{tot}Itot​.

  4. Vérifier avec les calculs théoriques.

2.1 Photorésistance (LDR)

Principe

Une LDR (Light Dependent Resistor) est une résistance qui diminue quand la lumière augmente.

  • Beaucoup de lumière → résistance faible → courant passe facilement.

  • Obscurité → résistance élevée → courant bloqué.

Câblage : pont diviseur LDR + résistance fixe

    │

   │R│  (10 kΩ fixe)
   └┬┘
    ●──── vers A0 (entrée analogique)
   ┌┴┐
   │ R_LDR │  (varie avec la lumière)
   └┬┘
    │
   GND

Logique de la mesure : la tension lue en A0 est UA0=5×RLDRR+RLDRU_{A0} = 5 \times \dfrac{R_{LDR}}{R + R_{LDR}}UA0​=5×R+RLDR​RLDR​​

  • Forte lumière → RLDRR_{LDR}RLDR​ faible → UA0U_{A0}UA0​ faible.

  • Obscurité → RLDRR_{LDR}RLDR​ grande → UA0U_{A0}UA0​ élevée.

2.2 Transistor NPN en commutation

Le transistor : un interrupteur commandé

3 broches : Collecteur (C), Base (B), Émetteur (E).

  • Bloqué (IB≈0I_B \approx 0IB​≈0) : aucun courant C→E, charge éteinte.

  • Saturé (un faible IBI_BIB​ suffit) : courant C→E passe, charge allumée.

Relations à connaître

IC=β×IBIE=IC+IBI_C = \beta \times I_B \qquad I_E = I_C + I_BIC​=β×IB​IE​=IC​+IB​

VBE≈0,7V(jonction B-E comme une diode)V_{BE} \approx 0{,}7\text{V} \quad\text{(jonction B-E comme une diode)}VBE​≈0,7V(jonction B-E comme une diode)

Méthode de calcul (saturation)

  1. Calculer IC,sat=VCCRCI_{C,sat} = \dfrac{V_{CC}}{R_C}IC,sat​=RC​VCC​​ (courant max dans le collecteur)

  2. Calculer IB,min=IC,satβI_{B,min} = \dfrac{I_{C,sat}}{\beta}IB,min​=βIC,sat​​ (courant de base minimum pour saturer)

  3. Calculer IBI_BIB​ réel =VBB−0,7RB= \dfrac{V_{BB} - 0{,}7}{R_B}=RB​VBB​−0,7​

  4. Si IB≥IB,minI_B \geq I_{B,min}IB​≥IB,min​ → saturé ✅ ; sinon → zone active (à éviter en commutation).

Exemple type (labo 5, β = 100)

Circuit : alimentation 5V (Arduino), RC=470ΩR_C = 470\text{Ω}RC​=470Ω (sur LED), RB=10kΩR_B = 10\text{kΩ}RB​=10kΩ, signal de base = 5V.

  • IC,sat=5−2470≈6,4mAI_{C,sat} = \dfrac{5 - 2}{470} \approx 6{,}4\text{mA}IC,sat​=4705−2​≈6,4mA (avec LED VF=2VV_F = 2\text{V}VF​=2V)

  • IB,min=6,4mA100=64µAI_{B,min} = \dfrac{6{,}4\text{mA}}{100} = 64\text{µA}IB,min​=1006,4mA​=64µA

  • IB=5−0,710,000=430µAI_B = \dfrac{5 - 0{,}7}{10{,}000} = 430\text{µA}IB​=10,0005−0,7​=430µA

  • 430µA≫64µA430\text{µA} \gg 64\text{µA}430µA≫64µA → transistor saturé ✅

Tout programme Arduino tient dans deux fonctions :

void setup() {

    // Configuration des broches, initialisation Serial, etc.

3.2 Entrées / Sorties numériques

Configuration

pinMode(broche, OUTPUT);   // sortie
pinMode(broche, INPUT);    // entrée

Lecture / écriture

3.3 Pull-up / Pull-down (LE piège à éviter)

Le problème : entrée flottante

Sans résistance, quand le BP est relâché, la broche n'est connectée à rien → elle "flotte" et peut lire n'importe quoi (HIGH ou LOW aléatoirement).

Solution 1 : Pull-down externe

   +5V ─── BP ─── broche Arduino ─── R(10kΩ) ─── GND

  • BP relâché → broche tirée vers GND → lecture LOW (0).

  • BP appuyé → broche au +5V → lecture HIGH (1).

Solution 2 : Pull-up externe

   +5V ─── R(10kΩ) ─── broche Arduino ─── BP ─── GND

  • BP relâché → broche tirée vers +5V → lecture HIGH (1).

  • BP appuyé → broche à GND → lecture LOW (0). ⚠️ Logique inversée.

Solution 3 : Pull-up INTERNE de l'Arduino (le plus simple ⭐)

pinMode(BP1, INPUT_PULLUP);

  • Plus besoin de résistance externe.

  • BP entre la broche et GND.

  • Lecture inversée : relâché = HIGH, appuyé = LOW.

if (digitalRead(BP1) == LOW) {
    // bouton appuyé
}

💡 À l'examen, utiliser INPUT_PULLUP simplifie le câblage (pas de résistance supplémentaire).

3.4 Entrées analogiques

Caractéristiques

  • 6 broches : A0 à A5.

  • Convertisseur ADC 10 bits : valeur entière 0 à 1023.

  • Plage de tension : 0V à 5V.

  • Résolution : 5V1023≈4,9mV\dfrac{5\text{V}}{1023} \approx 4{,}9\text{mV}10235V​≈4,9mV par unité.

Utilisation

int valeur = analogRead(A0);  // pas besoin de pinMode pour A0–A5
float tension = valeur * (5.0 / 1023.0);  // conversion en volts

La fonction map()

Très utile pour convertir une plage de valeurs en une autre :

int valeurAnalogique = analogRead(A0);          // 0 à 1023
int valeurPWM = map(valeurAnalogique, 0, 1023, 0, 255);
//                  valeur, vieuxMin, vieuxMax, nouveauMin, nouveauMax

3.5 Sorties analogiques (PWM)

Principe

Pas de vraie sortie analogique : on simule en variant le rapport cyclique (duty cycle) d'un signal carré.

  • Duty cycle 0% → sortie toujours à 0V (équivalent 0V analogique).

  • Duty cycle 50% → sortie HIGH la moitié du temps (équivalent 2,5V).

  • Duty cycle 100% → sortie toujours à 5V.

Broches PWM (marquées ~ sur la carte)

UNO : broches 3, 5, 6, 9, 10, 11.

Utilisation

pinMode(9, OUTPUT);
analogWrite(9, 128);  // valeur de 0 à 255 (correspond à 0 à 100% duty)

💡 PWM = 8 bits (0 à 255), alors que analogRead = 10 bits (0 à 1023). Pour piloter un PWM avec un analogRead, utiliser map().

4.1 Blink bloquant vs non-bloquant ⭐

Version bloquante (à éviter)

#define LED 13

void setup() {
    pinMode(LED, OUTPUT);
}

void loop() {
    digitalWrite(LED, HIGH);
    delay(500);   // ⚠️ BLOQUE tout le programme pendant 500ms
    digitalWrite(LED, LOW);
    delay(500);
}

Problème : pendant le delay(), impossible de lire un BP, faire autre chose, etc.

Version non-bloquante (avec millis()) — À MAÎTRISER ⭐⭐⭐

#define LED 13

unsigned long tempsPrecedent = 0;
const unsigned long INTERVALLE = 500;  // ms
bool etatLED = LOW;

void setup() {
    pinMode(LED, OUTPUT);
}

void loop() {
    unsigned long maintenant = millis();

    if (maintenant - tempsPrecedent >= INTERVALLE) {
        tempsPrecedent = maintenant;
        etatLED = !etatLED;              // inverse l'état
        digitalWrite(LED, etatLED);
    }

    // Ici on peut faire d'AUTRES choses (lire un BP, etc.)
}

💡 Principe millis() : renvoie le nombre de ms depuis le démarrage. On compare la différence avec un intervalle, sans jamais bloquer.

4.2 Télérupteur sans delay() ⭐⭐⭐

Cahier des charges : à chaque appui sur le BP, la LED change d'état. Pas de delay(). Anti-rebond intégré.

#define LED 13
#define BP1 2

bool etatLED = LOW;
bool etatBP_precedent = HIGH;        // HIGH car INPUT_PULLUP, BP relâché
unsigned long tempsDernierAppui = 0;
const unsigned long ANTI_REBOND = 50;  // ms

void setup() {
    pinMode(LED, OUTPUT);
    pinMode(BP1, INPUT_PULLUP);
    digitalWrite(LED, etatLED);
}

void loop() {
    bool etatBP_actuel = digitalRead(BP1);

    // Détection du FRONT DESCENDANT (relâché HIGH → appuyé LOW)
    if (etatBP_precedent == HIGH && etatBP_actuel == LOW) {
        // Vérifier qu'on n'est pas dans le rebond
        if (millis() - tempsDernierAppui > ANTI_REBOND) {
            etatLED = !etatLED;
            digitalWrite(LED, etatLED);
            tempsDernierAppui = millis();
        }
    }

    etatBP_precedent = etatBP_actuel;
}

Pourquoi ça marche

  1. Détection du front : on bascule uniquement quand le BP passe de relâché à appuyé, pas tant qu'il est maintenu enfoncé.

  2. Anti-rebond : un bouton mécanique génère des micro-impulsions parasites pendant ~10-50ms. On ignore tout nouveau changement pendant 50ms après le précédent.

  3. Pas de delay() : le programme reste réactif à autre chose.

4.3 Détection appui court / appui long ⭐⭐⭐

Pattern indispensable pour l'examen type. Mesure la durée d'appui et déclenche selon : court (<2s<2\text{s}<2s) ou long (≥2s\geq 2\text{s}≥2s).

#define LED 13
#define BP1 2

const unsigned long SEUIL_APPUI_LONG = 2000;  // 2 secondes
unsigned long tempsDebutAppui = 0;
bool bp_etait_appuye = false;

void setup() {
    pinMode(LED, OUTPUT);
    pinMode(BP1, INPUT_PULLUP);
    Serial.begin(9600);
}

void loop() {
    bool bp_appuye = (digitalRead(BP1) == LOW);  // INPUT_PULLUP → LOW = appuyé

    // ─── Front descendant : début d'appui ───
    if (bp_appuye && !bp_etait_appuye) {
        tempsDebutAppui = millis();
        bp_etait_appuye = true;
    }

    // ─── Front montant : relâchement → on analyse la durée ───
    if (!bp_appuye && bp_etait_appuye) {
        unsigned long duree = millis() - tempsDebutAppui;
        bp_etait_appuye = false;

        if (duree < SEUIL_APPUI_LONG) {
            Serial.print("Appui bref - duree: ");
            Serial.print(duree);
            Serial.println(" ms");
            // Action appui court (ex. télérupteur)
        } else {
            Serial.print("Appui long - duree: ");
            Serial.print(duree);
            Serial.println(" ms");
            // Action appui long (ex. déclenchement clignotement)
        }
    }
}

💡 Astuce examen : on peut combiner ce pattern avec le télérupteur (4.2) en remplaçant l'action "court" par l'inversion de la LED.

4.4 Clignotement à fréquence précise — calcul T ↔ f

Relations à connaître

f=1TT=1ff = \dfrac{1}{T} \qquad T = \dfrac{1}{f}f=T1​T=f1​

Pour un duty cycle de 50%, on alterne HIGH/LOW toutes les demi-périodes :

intervalle=T2=12f\text{intervalle} = \dfrac{T}{2} = \dfrac{1}{2f}intervalle=2T​=2f1​

Tableau de référence fréquence ↔ période

Rappel : T (ms) = 1000 / f (Hz) — Demi-période = T / 2

  • f = 2 Hz → T = 500 ms → demi-période = 250 ms

  • f = 1 Hz → T = 1000 ms → demi-période = 500 ms

  • f = 0,8 Hz → T = 1250 ms → demi-période = 625 ms

  • f = 0,6 Hz → T = 1666 ms → demi-période = ≈ 833 ms ← examen type

  • f = 0,5 Hz → T = 2000 ms → demi-période = 1000 ms

Code : 2 LEDs à des fréquences différentes (labo 6 manip 5)

#define LED1 12
#define LED2 13

const unsigned long DEMI_PERIODE_1 = 500;   // 1 Hz
const unsigned long DEMI_PERIODE_2 = 625;   // 0,8 Hz

unsigned long t1_precedent = 0;
unsigned long t2_precedent = 0;
bool etat1 = LOW;
bool etat2 = LOW;

void setup() {
    pinMode(LED1, OUTPUT);
    pinMode(LED2, OUTPUT);
}

void loop() {
    unsigned long maintenant = millis();

    if (maintenant - t1_precedent >= DEMI_PERIODE_1) {
        t1_precedent = maintenant;
        etat1 = !etat1;
        digitalWrite(LED1, etat1);
    }

    if (maintenant - t2_precedent >= DEMI_PERIODE_2) {
        t2_precedent = maintenant;
        etat2 = !etat2;
        digitalWrite(LED2, etat2);
    }
}

4.6 Dimmer : potentiomètre → PWM sur LED

#define POT_PIN A0

    pinMode(LED_PIN, OUTPUT);
}

}

Q1 — Circuit LED + résistance ⚙️

   +5V (Arduino) ─── R ─── (anode) LED (cathode) ─── GND

UR=Vsource−VLED=5−2=3VU_R = V_{source} - V_{LED} = 5 - 2 = 3\text{V}UR​=Vsource​−VLED​=5−2=3V

Code complet et commenté

// ════════════════════════════════════════════════════════
//  Examen Systèmes Embarqués - Programme complet
//  Q2 : télérupteur sur LED1
//  Q3 : appui long >= 2s → les 2 LEDs clignotent à 0,6 Hz
//  Q4 : affichage type d'appui + période mesurée
// ════════════════════════════════════════════════════════

#define BP1   2
#define LED1  12
#define LED2  13

// ─── Paramètres ───
const unsigned long SEUIL_LONG = 2000;   // 2 secondes pour un appui long
const unsigned long ANTI_REBOND = 50;    // anti-rebond en ms
const float FREQUENCE_HZ = 0.6;          // 0,6 Hz demandé à l'examen
// Demi-période en ms pour duty cycle 50% : T/2 = 1/(2*f) en secondes
const unsigned long DEMI_PERIODE = (unsigned long)(1000.0 / (2.0 * FREQUENCE_HZ));
// → DEMI_PERIODE ≈ 833 ms

// ─── États ───
bool etatLED1 = LOW;
bool etatLED2 = LOW;
bool modeClignotement = false;   // mis à true par un appui long

// ─── Détection appui ───
bool bp_etait_appuye = false;
unsigned long tempsDebutAppui = 0;

// ─── Clignotement ───
unsigned long tempsDerniereBascule = 0;
unsigned long tempsBasculePrecedente = 0;  // pour mesurer la période

void setup() {
    pinMode(BP1, INPUT_PULLUP);
    pinMode(LED1, OUTPUT);
    pinMode(LED2, OUTPUT);
    digitalWrite(LED1, LOW);
    digitalWrite(LED2, LOW);
    Serial.begin(9600);
    Serial.println("=== Demarrage ===");
}

void loop() {
    // ────────────────────────────────────────
    // 1. Gestion du bouton poussoir
    // ────────────────────────────────────────
    bool bp_appuye = (digitalRead(BP1) == LOW);

    // Front descendant : début d'appui
    if (bp_appuye && !bp_etait_appuye) {
        if (millis() - tempsDebutAppui > ANTI_REBOND) {
            tempsDebutAppui = millis();
            bp_etait_appuye = true;
        }
    }

    // Front montant : relâchement → on analyse la durée
    if (!bp_appuye && bp_etait_appuye) {
        unsigned long duree = millis() - tempsDebutAppui;
        bp_etait_appuye = false;

        if (duree < SEUIL_LONG) {
            // ── APPUI BREF : télérupteur sur LED1 ──
            Serial.print("Appui bref - duree: ");
            Serial.print(duree);
            Serial.println(" ms");

            // On arrête le clignotement s'il était actif
            modeClignotement = false;
            digitalWrite(LED2, LOW);
            etatLED2 = LOW;

            // Télérupteur LED1
            etatLED1 = !etatLED1;
            digitalWrite(LED1, etatLED1);
        }
        else {
            // ── APPUI LONG : déclenchement clignotement 2 LEDs ──
            Serial.print("Appui long - duree: ");
            Serial.print(duree);
            Serial.println(" ms");

            modeClignotement = true;
            tempsDerniereBascule = millis();
            tempsBasculePrecedente = millis();
        }
    }

    // ────────────────────────────────────────
    // 2. Mode clignotement (LED1 + LED2 à 0,6 Hz)
    // ────────────────────────────────────────
    if (modeClignotement) {
        unsigned long maintenant = millis();

        if (maintenant - tempsDerniereBascule >= DEMI_PERIODE) {
            // Calcul de la période mesurée (un cycle = 2 bascules)
            unsigned long periode_ms = (maintenant - tempsBasculePrecedente) * 2;
            float periode_s = periode_ms / 1000.0;

            // Bascule des 2 LEDs ensemble
            etatLED1 = !etatLED1;
            etatLED2 = !etatLED2;
            digitalWrite(LED1, etatLED1);
            digitalWrite(LED2, etatLED2);

            // Affichage de la période mesurée
            Serial.print("Periode mesuree : ");
            Serial.print(periode_s, 3);   // 3 décimales
            Serial.println(" s");

            tempsBasculePrecedente = tempsDerniereBascule;
            tempsDerniereBascule = maintenant;
        }
    }
}

Points-clés du code à comprendre

  1. INPUT_PULLUP sur le BP → pas de résistance externe nécessaire, logique inversée (LOW = appuyé).

  2. Détection de fronts (bp_appuye && !bp_etait_appuye) → on ne réagit qu'au changement d'état, pas tant qu'on maintient.

  3. millis() partout, JAMAIS delay() → cahier des charges respecté.

  4. DEMI_PERIODE calculé depuis la fréquence demandée (0,6 Hz) → adaptable si la consigne change.

  5. Mesure de période = différence de temps entre deux bascules × 2 (un cycle complet = 2 bascules).

  6. Affichage série structuré : tag descriptif + valeur + unité.

🔥 Astuces de dernière minute

Si quelque chose ne marche pas

LED ne s'allume pas → Sens inversé ou résistance trop grande — vérifier l'anode (longue patte), mesurer au multimètre

BP toujours actif / comportement aléatoire → Entrée flottante — utiliser INPUT_PULLUP

BP déclenche plusieurs fois (rebond) → Pas d'anti-rebond — ajouter le test millis() - dernier > 50

Code ne s'upload pas → Mauvais port COM ou mauvaise carte — Outils → Port / Type de carte

Serial n'affiche rien / caractères bizarres → Mauvais baud rate — vérifier que les deux côtés sont à 9600

LED clignote trop vite → delay() trop court ou condition millis() mal écrite — vérifier la logique de bascule

Vérifications avant de rendre

  • sans erreur.

  • dans le code (vérifier par recherche).

  • fonctionne (1 appui = 1 changement, pas plusieurs).

  • déclenche bien le clignotement.

  • clignotent ensemble à la bonne fréquence.

  • affiche bien le type d'appui ET la période mesurée.

  • est propre, pas de fil qui dépasse.

  • sur chaque LED est présente.

C. Pièges classiques à l'examen ⚠️

  1. Oublier la résistance sur la LED → claquage → –6 pts.

  2. Utiliser int pour millis() → débordement après ~32 secondes → bug imprévisible. Toujours unsigned long.

  3. delay() interdit dans les cahiers des charges → utiliser millis().

  4. Confondre analogRead (0–1023) et analogWrite (0–255) → utiliser map() pour relier les deux.

  5. Pull-up/pull-down oublié → BP imprévisible. Utiliser INPUT_PULLUP.

  6. Logique inversée avec INPUT_PULLUP : LOW = appuyé, HIGH = relâché.

  7. Anti-rebond oublié sur le BP → télérupteur déclenche 2-3 fois.

  8. Détection sur état au lieu de front → action répétée tant que le BP est maintenu.

  9. Pas de Serial.begin() dans setup → moniteur série affiche rien / des caractères bizarres.

  10. Broches 0 et 1 utilisées → upload bloqué.

  11. Mauvais baud rate : Arduino à 9600, moniteur à 115200 → caractères incompréhensibles.

  12. Variables non remises à zéro entre les états → comportement incohérent.
