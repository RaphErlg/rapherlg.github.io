1. Démarche universelle pour tout labo

Cette démarche s'applique à TOUS les labos et à l'examen :

1. LIRE l'énoncé en entier (2 fois si nécessaire)
2. DESSINER la topologie sur papier si pas fournie
3. CALCULER tous les sous-réseaux (subnetting)
4. PLANIFIER l'adressage (qui a quelle IP)
5. COUCHE 1 (physique)  : câbler dans Packet Tracer
6. COUCHE 2 (liaison)   : VLANs sur les switches + trunks
7. COUCHE 3 (réseau)    : 
   a. IPs sur routeurs + no shutdown
   b. Routage interne (statique ou RIP)
   c. Routes par défaut + distribution
8. NAT (sur routeur de bordure)
9. SERVICES (DHCP, HTTP, DNS)
10. PCs (IP + masque + gateway + DNS)
11. TESTER étape par étape :
    intra-VLAN → inter-VLAN → inter-LAN → Internet
12. RÈGLES DE L'ART (hostname, banner, passwords, descriptions)
13. SAUVEGARDER (copy run start sur tous les équipements)

Règle d'or : ne saute aucune étape. C'est plus rapide de faire dans l'ordre que de courir après les bugs.

2. Les modes Cisco IOS

Router>                        Mode USER (utilisateur)
                                   - Très limité
                                   - Quelques "show"
   ↓ enable
Router#                        Mode PRIVILEGED (privilégié)
                                   - Tous les "show"
                                   - Pas encore de config
   ↓ configure terminal
Router(config)#                Mode CONFIG GLOBAL
                                   - Hostname, routes, mots de passe
   ↓ interface g0/0
Router(config-if)#             Mode CONFIG INTERFACE
                                   - IP, no shutdown, description

Pour sortir :

  • exit : remonter d'un niveau

  • end ou Ctrl+Z : retour direct au mode #

  • disable : retour mode >

Mémo : > = je regarde | # = je peux tout faire | (config)# = je modifie globalement | (config-if)# = je modifie une interface précise.

3. Reset des équipements

Routeur

Router> enable
Router# erase startup-config
Router# reload

Switch (⚠️ une étape EN PLUS)

Switch> enable
Switch# erase startup-config
Switch# delete vlan.dat              ← OBLIGATOIRE sinon les VLANs survivent !
Switch# reload

Récupération mot de passe (mode ROMmon)

1. Power off / Power on
2. Pendant le démarrage : Ctrl+Break
3. Rommon> confreg 0x2142     ← démarre SANS startup-config
4. Rommon> reset
5. (config vide, modifie ce que tu veux)
6. Router(config)# config-register 0x2102  ← restaure démarrage normal
7. Router# copy run start
8. Router# reload

Les 2 formules à connaître par cœur

  • Nombre de sous-réseaux = 2ⁿ (n = bits empruntés)

5. Configuration interfaces

Interface routeur classique

R1(config)# interface gigabitEthernet 0/0
R1(config-if)# description To Switch SW1
R1(config-if)# ip address 192.168.1.1 255.255.255.0
R1(config-if)# no shutdown                      ← OBLIGATOIRE

Interface série Smart Serial (labo 03)

R1(config)# interface serial 0/0/0
R1(config-if)# ip address 192.168.4.1 255.255.255.0
R1(config-if)# clock rate 64000                 ← UNIQUEMENT côté DCE
R1(config-if)# no shutdown

DCE = côté qui fournit le signal d'horloge (visible avec un pictogramme d'horloge dans Packet Tracer).

Interface loopback (simulation LAN ou Internet)

R1(config)# interface loopback 1
R1(config-if)# ip address 8.8.8.8 255.255.255.255
                                                 ← pas besoin de no shutdown !

Sous-interfaces VLAN (Router-on-a-stick, labo 04+)

R1(config)# interface gigabitEthernet 0/0
R1(config-if)# no shutdown
R1(config-if)# exit

R1(config)# interface gigabitEthernet 0/0.10
R1(config-subif)# encapsulation dot1q 10        ← n° du VLAN, doit matcher !
R1(config-subif)# ip address 192.168.1.1 255.255.255.128

R1(config)# interface gigabitEthernet 0/0.20
R1(config-subif)# encapsulation dot1q 20
R1(config-subif)# ip address 192.168.1.129 255.255.255.128

Sauvegarde config (à faire SYSTÉMATIQUEMENT)

Router# copy running-config startup-config
                  ↑                   ↑
            actuelle (RAM)       démarrage (NVRAM)

! Raccourci :
Router# copy run start

6. VLANs

Création des VLANs sur un switch

SW1(config)# vlan 10
SW1(config-vlan)# name Users10
SW1(config-vlan)# exit
SW1(config)# vlan 20
SW1(config-vlan)# name Users20
SW1(config-vlan)# exit
SW1(config)# vlan 99
SW1(config-vlan)# name Management

Port en mode access (vers PC, imprimante…)

SW1(config)# interface fastEthernet 0/1
SW1(config-if)# switchport mode access
SW1(config-if)# switchport access vlan 10
SW1(config-if)# switchport nonegotiate           ← désactive DTP (sécurité)

Pour plusieurs ports d'un coup :

SW1(config)# interface range fa0/1 - 5
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 10
SW1(config-if-range)# switchport nonegotiate

Port en mode trunk (vers routeur, autre switch, serveur 802.1q)

SW1(config)# interface gigabitEthernet 0/1
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan 10,20,99   ← restriction recommandée
SW1(config-if)# switchport trunk native vlan 99          ← bonne pratique sécurité

SVI (interface virtuelle pour management du switch)

SW1(config)# interface vlan 99
SW1(config-if)# ip address 192.168.99.10 255.255.255.0
SW1(config-if)# no shutdown
SW1(config-if)# exit
SW1(config)# ip default-gateway 192.168.99.1   ← pour répondre hors VLAN

Vérification

SW1# show vlan brief                          ← VLANs et leurs ports
SW1# show interfaces trunk                    ← détails sur les trunks
SW1# show interfaces fa0/1 switchport         ← détails d'un port

7. Routage statique

Syntaxe générale

R1(config)# ip route <réseau destination> <masque> <next-hop>

Exemples

! Route vers un réseau distant
R1(config)# ip route 192.168.2.0 255.255.255.0 192.168.10.2

! Route par défaut (TOUTES les destinations inconnues)
R1(config)# ip route 0.0.0.0 0.0.0.0 80.70.60.1

! Route flottante (backup, distance admin élevée)
R1(config)# ip route 192.168.10.0 255.255.255.0 192.168.5.2 200
                                                              ↑
                                                   distance admin (défaut: 1)

Lecture de la table de routage

Codes: C - connected, S - static, R - RIP, *  - candidate default

C    192.168.1.0/24    is directly connected, GigabitEthernet0/0
S    192.168.2.0/24    [1/0] via 192.168.10.2
S*   0.0.0.0/0         [1/0] via 80.70.60.1                ← route par défaut
R    192.168.4.0/24    [120/2] via 192.168.10.2            ← route apprise par RIP
R*   0.0.0.0/0         [120/1] via 192.168.10.1            ← route par défaut RIP

Les 3 principes fondamentaux

  1. Chaque routeur décide tout seul (ne consulte pas les autres)

  2. Vue partielle (ne connaît pas la topologie globale)

  3. Aller-retour pas garanti (penser à la route retour : c'est l'erreur n°1)

9. NAT (Network Address Translation)

Configuration NAT Overload (PAT) — le cas standard

! Étape 1 : ACL pour définir QUI peut être traduit
R1(config)# ip access-list standard 1
R1(config-std-nacl)# permit 192.168.1.0 0.0.0.255          ← tout le /24
R1(config-std-nacl)# exit

! ⚠️ Wildcard mask = inverse du masque !
! /24 (255.255.255.0)  → wildcard 0.0.0.255
! /27 (255.255.255.224) → wildcard 0.0.0.31

! Étape 2 : Activer le NAT
R1(config)# ip nat inside source list 1 interface gigabitEthernet 0/1 overload
                                                            ↑           ↑
                                                       interface WAN   PAT

! Étape 3 : Désigner les interfaces
R1(config)# interface gigabitEthernet 0/1
R1(config-if)# ip nat outside              ← interface WAN (vers FAI)
R1(config-if)# exit

R1(config)# interface gigabitEthernet 0/0.10
R1(config-if)# ip nat inside               ← interfaces LAN (chaque sous-int VLAN)

R1(config)# interface gigabitEthernet 0/0.20
R1(config-if)# ip nat inside

! Etc. pour chaque sous-interface interne

Vérification

R1# show ip nat translations             ! voir les translations en cours
R1# show ip nat statistics                ! statistiques NAT
R1# clear ip nat translation *            ! vider la table (pour debug)
R1# debug ip nat                          ! voir le NAT en temps réel

Les 3 types de NAT

Type

Mapping

Usage

Statique

1 IP privée ↔ 1 IP publique fixe

Exposer un serveur interne

Dynamique

Pool d'IPs publiques

Plusieurs IPs publiques disponibles

Overload (PAT)

Toutes IPs privées → 1 IP publique (ports différents)

90% des cas (box maison)

Effet immédiat

Sans NAT : le FAI a besoin de routes vers les IPs privées (jamais en vrai !). Avec NAT : le FAI ne connaît que ton IP publique. Plus simple, plus sécurisé, plus réaliste.

10. Distribution route par défaut via RIP

Le combo gagnant : statique vers le FAI + RIP en interne + propagation auto.

Sur le routeur de bordure

! Route statique par défaut vers le FAI
R-Bordure(config)# ip route 0.0.0.0 0.0.0.0 70.70.70.1

! RIP avec distribution
R-Bordure(config)# router rip
R-Bordure(config-router)# version 2
R-Bordure(config-router)# no auto-summary
R-Bordure(config-router)# network 192.168.10.0      ← liens internes UNIQUEMENT
R-Bordure(config-router)# default-information originate    ← ✨ LA COMMANDE CLÉ

Résultat sur les routeurs internes

R*   0.0.0.0/0 [120/1] via 192.168.10.1
                                          ↑
                              R = appris par RIP
                              * = candidate default

Les routeurs internes reçoivent automatiquement la route par défaut. Plus besoin de la configurer en statique sur chacun !

Règle absolue

NE JAMAIS annoncer le réseau du FAI dans RIP (network 70.70.70.0 interdit). Sinon : leak d'infos internes vers le FAI + pollution.

11. IPv6

Format et simplification

Forme complète : 2001:0DB8:ABCD:0001:0000:0000:0000:0001
Règle 1 (zéros initiaux) : 2001:DB8:ABCD:1:0:0:0:1
Règle 2 (compression ::) : 2001:DB8:ABCD:1::1

⚠️ :: une seule fois par adresse (sinon ambigu).

12. Services : DHCP, HTTP, DNS

DHCP (Packet Tracer Server-PT)

  1. Server-PT → Onglet Services → DHCP

  2. Renseigner :

    • Pool name : ex. LAN_VLAN10

    • Default Gateway : IP du routeur côté ce VLAN

    • DNS Server : IP du serveur DNS interne

    • Start IP Address : ex. 192.168.1.10

    • Subnet Mask : ex. 255.255.255.128

    • Maximum users : nombre d'IPs disponibles

  3. Save + Service ON

Sur les PCs : passer en DHCP dans IP Configuration.

HTTP (serveur web Packet Tracer)

  1. Server-PT → Onglet Services → HTTP

  2. Activer HTTP (et HTTPS si voulu)

  3. Personnaliser index.html si besoin

Test : sur PC → onglet Desktop → Web Browser → IP ou URL.

DNS (serveur DNS Packet Tracer)

  1. Server-PT → Onglet Services → DNS

  2. DNS Service : ON

  3. Resource Record :

    • Name : www.test.com

    • Type : A Record

    • Address : 80.70.60.10 (IP du serveur web)

  4. Add

Sur les PCs : renseigner DNS Server dans IP Configuration.

Test :

PC> nslookup www.test.com
PC> ping www.test.com
PC> (Web Browser) http://www.test.com

13. Management IT et SSH

Configuration SSH (à connaître par cœur)

! 1. Hostname et nom de domaine (REQUIS pour les clés RSA)
SW1(config)# hostname SW1
SW1(config)# ip domain-name labo.local

! 2. Génération des clés RSA
SW1(config)# crypto key generate rsa
                How many bits in the modulus [512]: 1024

! 3. Compte local
SW1(config)# username admin secret cisco
SW1(config)# enable secret class

! 4. Lignes VTY (telnet/SSH distant)
SW1(config)# line vty 0 4
SW1(config-line)# transport input ssh        ← SSH only (pas telnet)
SW1(config-line)# login local                ← utiliser usernames locaux
SW1(config-line)# exec-timeout 10            ← déco auto après 10 min
SW1(config-line)# exit

! 5. Forcer SSH v2 (plus sécurisé)
SW1(config)# ip ssh version 2

Test depuis un PC dans Packet Tracer

PC> Command Prompt
PC> ssh -l admin 192.168.99.10
Password: cisco
SW1>

Mots de passe imposés (cohérence labos)

  • Console et VTY : cisco

  • Enable secret : class

14. Règles de l'art Cisco

À appliquer systématiquement sur chaque équipement.

! Hostname significatif
Router(config)# hostname R-Bruxelles

! Pas de recherche DNS (évite délais sur les fautes de frappe)
R-Bruxelles(config)# no ip domain-lookup

! Banner d'avertissement
R-Bruxelles(config)# banner motd % This is a secured system. Authorized Access ONLY !!! %

! Mot de passe console
R-Bruxelles(config)# line console 0
R-Bruxelles(config-line)# password cisco
R-Bruxelles(config-line)# login
R-Bruxelles(config-line)# logging synchronous   ← logs ne coupent pas la saisie
R-Bruxelles(config-line)# exec-timeout 15
R-Bruxelles(config-line)# exit

! Mot de passe enable secret (chiffré)
R-Bruxelles(config)# enable secret class

! Chiffrer TOUS les mots de passe (y compris ceux en clair)
R-Bruxelles(config)# service password-encryption

! Description sur chaque interface utilisée
R-Bruxelles(config)# interface g0/0
R-Bruxelles(config-if)# description To Switch SW1

! Désactiver les interfaces non-utilisées
R-Bruxelles(config)# interface g0/2
R-Bruxelles(config-if)# shutdown

Checklist règles de l'art

  • ☐ hostname significatif

  • ☐ no ip domain-lookup

  • ☐ banner motd

  • ☐ password console

  • ☐ enable secret

  • ☐ service password-encryption

  • ☐ description sur interfaces utilisées

  • ☐ shutdown sur interfaces non-utilisées

  • ☐ SSH configuré (si management requis)

  • ☐ copy run start

15. Diagnostic et debug

La séquence de tests (TOUJOURS dans cet ordre)

1. Intra-sous-réseau     : PC1 → autre PC du même VLAN
2. PC → sa gateway       : PC1 → IP du routeur
3. Inter-VLAN intra-site : PC VLAN 10 → PC VLAN 20
4. Inter-site            : PC LAN 1 → PC LAN 2
5. Internet par IP       : PC → 8.8.8.8
6. Internet par nom      : PC → www.test.com (test DNS)

Si une étape échoue, arrête-toi là et débogue avant d'avancer.

Les 5 commandes de diagnostic essentielles

! État de TOUTES les interfaces (UP/DOWN, IP)
Router# show ip interface brief

! Table de routage complète
Router# show ip route

! Configuration en cours (RAM)
Router# show running-config

! Détails sur un protocole de routage
Router# show ip protocols

! Test ping/traceroute
Router# ping 192.168.1.10
Router# traceroute 192.168.1.10

Sur un switch en plus

SW1# show vlan brief
SW1# show interfaces trunk
SW1# show interfaces fa0/1 switchport
SW1# show mac address-table

Pour le NAT

R1# show ip nat translations
R1# show ip nat statistics
R1# clear ip nat translation *

Pour IPv6

R1# show ipv6 interface brief
R1# show ipv6 route
R1# show ipv6 neighbors

Méthode "5 questions" quand un ping échoue

  1. L'interface est-elle UP ? → show ip interface brief

  2. L'IP est-elle correcte ? → vérifier dans la conf

  3. Le routeur connaît-il le réseau destination ? → show ip route

  4. Le retour est-il configuré ? → vérifier la table de routage du routeur distant

  5. Les ACL ou NAT bloquent-ils ? → show ip nat translations / ACLs

16. Top 30 des pièges récurrents

Configuration de base

  1. ❌ Oublier no shutdown sur une interface → reste DOWN

  2. ❌ Oublier copy run start → tout perdu au reload

  3. ❌ Oublier delete vlan.dat au reset switch → VLANs ressuscitent

  4. ❌ Oublier la gateway sur un PC → ping local OK, distant KO

  5. ❌ Confondre running-config et startup-config

Subnetting et adressage

  1. ❌ Oublier le −2 dans le calcul des hôtes (réseau + broadcast)

  2. ❌ Mettre une IP dans la plage réservée (.0 réseau ou .255 broadcast)

  3. ❌ Confondre /25, /26, /27 dans les commandes

  4. ❌ Mauvais wildcard (255 - X !)

VLANs

  1. ❌ Oublier encapsulation dot1q X sur les sous-interfaces

  2. ❌ Numéro dot1q ≠ numéro de VLAN sur le switch

  3. ❌ Oublier switchport mode access (port reste en dynamic auto)

  4. ❌ Mettre des PCs dans le VLAN 1

  5. ❌ Oublier ip default-gateway sur le switch pour SSH inter-VLAN

Routage

  1. ❌ Oublier la route retour (l'erreur n°1 du routage statique !)

  2. ❌ Mettre une route statique pour un réseau directement connecté

  3. ❌ Mauvais next-hop dans ip route

  4. ❌ RIP : oublier no auto-summary avec VLSM

  5. ❌ RIP : annoncer un réseau distant dans network

  6. ❌ Tester avant la convergence (~30 secondes pour RIP)

NAT

  1. ❌ Wildcard mask au lieu du masque réseau dans l'ACL

  2. ❌ Oublier overload → mode "1 translation à la fois"

  3. ❌ Oublier ip nat inside sur toutes les sous-interfaces

  4. ❌ Oublier ip nat outside sur l'interface WAN

Distribution route (labo 09 et 10)

  1. ❌ Oublier default-information originate → routes par défaut bloquées

  2. ❌ Annoncer le réseau du FAI dans RIP (interdit !)

IPv6

  1. ❌ 🚨 Oublier ipv6 unicast-routing sur le routeur (le piège n°1 d'IPv6)

  2. ❌ Utiliser link-local comme next-hop sans préciser l'interface

  3. ❌ Compression :: utilisée 2 fois dans une adresse

SSH

  1. ❌ Oublier crypto key generate rsa avant transport input ssh

17. Mémo de commandes ultra-condensé

Commandes "10 secondes" pour gagner du temps

! Entrée mode config rapide
en
conf t

! Configuration interface rapide
int g0/0
ip add 192.168.1.1 255.255.255.0
no sh
exit

! Sauvegarde
do wr        ← raccourci pour copy run start
copy r s

! Voir l'état des interfaces
do sh ip int br      ← depuis n'importe quel mode

! Voir la table de routage
do sh ip route

! Sortir directement au mode #
end

Configuration d'une interface VLAN en 4 lignes

int g0/0.10
encap dot1q 10
ip add 192.168.1.1 255.255.255.128
exit

NAT minimal (à connaître par cœur)

! ACL
ip access-list standard 1
permit 192.168.0.0 0.0.255.255

! NAT
ip nat inside source list 1 interface g0/1 overload

! Sur interfaces
int g0/1
ip nat outside

int g0/0.10
ip nat inside

Configuration RIP minimale

router rip
version 2
no auto-summary
network 192.168.1.0
network 192.168.10.0
default-information originate     ← si tu es le routeur de bordure

Activation IPv6 en 3 lignes

ipv6 unicast-routing       ← À NE PAS OUBLIER

int g0/0
ipv6 add 2001:DB8::1/64
ipv6 add FE80::1 link-local
no sh
