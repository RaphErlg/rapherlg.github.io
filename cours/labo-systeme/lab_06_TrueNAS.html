Lab_06_TrueNAS


TrueNAS — Fiche de procédure autonome

1. But

Mettre en place un NAS TrueNAS dans VirtualBox, puis configurer : le stockage ZFS, les utilisateurs, les permissions, les partages SMB, un disque iSCSI et un partage WebDAV sécurisé.

2. Architecture utilisée

VM TrueNAS

  • Type : BSD / FreeBSD 64 bits.

  • RAM : 2 Go pour le labo (8 Go recommandés en usage réel).

  • Disques :

    • 1 disque système : 16 Go minimum.

    • 3 disques de données : 20 Go chacun.

  • Réseau :

    • Carte 1 : Réseau interne.

    • Carte 2 : NAT ou pont.

VM cliente

  • Windows 10 Entreprise sur le même réseau interne que TrueNAS.

Idée clé : Le disque système sert à TrueNAS, les autres disques servent aux données.

3. Installation de TrueNAS

  1. Créer la VM dans VirtualBox.

  2. Monter l’ISO TrueNAS.

  3. Démarrer la VM et choisir Install/Upgrade.

  4. Installer le système sur le disque de 16 Go uniquement.

  5. Définir le mot de passe root (exemple : tttttt).

  6. Retirer l’ISO après installation.

  7. Redémarrer la VM.

4. Réseau TrueNAS

Depuis le menu console bleu de TrueNAS :

Interface 1 (réseau interne)

  • Configurer l'IPv4 manuellement : 172.16.20.200/24

Interface 2 (NAT/pont)

  • Activer le DHCP.

Quand tout est correct, l’interface Web est accessible sur un navigateur via : http://172.16.20.200

5. Réseau Windows 10 (Client)

Configurer la carte réseau (Ethernet) manuellement dans Windows :

  • Adresse IP : 172.16.20.11

  • Longueur du préfixe de sous-réseau : 24 (ne pas mettre 255.255.255.0)

  • Passerelle : laisser vide ou mettre 172.16.20.254

6. Premiers réglages dans l’interface Web

Connexion avec l'utilisateur root et votre mot de passe.

Dans System > General :

  • Console Keyboard Map : Belgian.

  • Timezone : Europe/Brussels.

  • Cocher "Show console messages" dans les options avancées.

7. Création du pool ZFS (Stockage)

Menu : Storage > Pools > Add > Create new pool

  • Nom du pool : Pool1

  • Disques : Sélectionner les 3 disques de 20 Go.

  • Type : Raid-z (pour la redondance).

  • Cliquer sur Create.

8. Création des datasets (Dossiers)

Dans Pool1, cliquer sur les trois petits points > Add Dataset.

Créer 5 Datasets avec des quotas dans les "Advanced Options" :

  • Coco (Quota : 2 GiB)

  • Momo (Quota : 2 GiB)

  • Nico (Quota : 2 GiB)

  • winshare (Quota : 5 GiB)

  • public (Quota : 8 GiB)

9. Groupes et Utilisateurs

Les Groupes

Menu : Accounts > Groups > Add

  • winusers (GID : 1001)

  • Allusers (GID : 1002)

Les Utilisateurs

Menu : Accounts > Users > Add

  • coco : Groupes auxiliaires winusers + Allusers -> Home Directory : /mnt/Pool1/Coco

  • momo : Groupes auxiliaires winusers + Allusers -> Home Directory : /mnt/Pool1/Momo

  • nico : Groupe auxiliaire Allusers (pas winusers) -> Home Directory : /mnt/Pool1/Nico

(Note : TrueNAS crée aussi un groupe personnel caché pour chaque utilisateur, c'est normal).

10. Permissions des datasets (ACL/Droits)

Cliquer sur les trois petits points du Dataset > Edit Permissions.

Dossiers privés (Coco, Momo, Nico)

  • User : L'utilisateur lui-même (ex: coco).

  • Droits : Cocher Read/Write/Execute uniquement pour la ligne "User". Décochez tout pour Group et Other.

Dossier partagé (winshare)

  • User : nobody

  • Group : winusers

  • Droits : Cocher Read/Write/Execute pour la ligne "Group". Décochez tout pour Other.

Dossier ouvert (public)

  • User : nobody

  • Group : Allusers

  • Droits : Read/Write/Execute pour "Group". Cocher seulement Read/Execute pour "Other".

11. Partages Windows (SMB)

Menu : Sharing > Windows Shares (SMB) > Add

Créer 4 partages :

  • Coco pointant vers /mnt/Pool1/Coco

  • Momo pointant vers /mnt/Pool1/Momo

  • Winshare pointant vers /mnt/Pool1/winshare

  • Public pointant vers /mnt/Pool1/public

Réglages spécifiques :

  • Pour le partage Public : Aller dans Advanced Options et cocher Allow Guest Access.

  • Pour masquer les fichiers systèmes sous Windows, ajouter ceci dans les Auxiliary Parameters (si disponible) : veto files = /Temporary Items/.DS_Store/.AppleDB/.profile/.shrc delete veto files = yes

12. Accéder aux partages depuis Windows 10

Pour éviter les bugs de l'icône "Réseau" de Windows :

  1. Ouvrir l'Explorateur de fichiers > Ce PC.

  2. Cliquer sur Connecter un lecteur réseau.

  3. Taper le chemin : \\172.16.20.200\Coco (ou Winshare, etc.).

  4. Cocher absolument "Se connecter à l'aide de nom d'utilisateur différents".

  5. S'identifier avec coco et le mot de passe du NAS.

(Astuce : Si le compte local Windows s'appelle "coco" avec le même mot de passe que le NAS, l'accès est automatique).

13. Stockage en mode bloc (iSCSI)

Objectif : Présenter un disque virtuel brut à Windows.

  1. Créer le volume : Storage > Pools > Add Zvol. Nom : Disque_iSCSI, Taille : 5 GiB.

  2. Activer le service : Menu Services > Démarrer iSCSI et cocher le lancement automatique.

  3. Configurer iSCSI (Sharing > Block Shares iSCSI) :

    • Portals : Ajouter l'IP 172.16.20.200 sur le port 3260.

    • Initiators : Cocher "Allow All Initiators".

    • Targets : Ajouter une cible (ex: truenastarget).

    • Extents : Ajouter un "Device" (et non File) pointant vers le Zvol créé (ex: extent1).

    • Associated Targets : Lier la Target à l'Extent.

  4. Côté Windows : Ouvrir "Initiateur iSCSI", taper 172.16.20.200, cliquer sur Connexion rapide. Ensuite, ouvrir "Gestion des disques", initialiser et formater le nouveau disque détecté.

14. Accès Web sécurisé (WebDAV + Certificats)

1. Autorité de Certification (CA)

Menu : System > CAs > Add

  • Nom : TrueNAS_Autority

  • Type : Internal CA

  • Clé : RSA 2048

  • Digest : SHA256

2. Certificat SSL

Menu : System > Certificates > Add

  • Nom : TrueNas_certificate

  • Type : Internal Certificate

  • Signé par : TrueNAS_Autority

  • Common Name : example.com (ou l'IP du serveur)

3. Droits pour WebDAV

WebDAV utilise un utilisateur interne nommé webdav.

  • Aller dans Accounts > Groups.

  • Modifier les membres du groupe winusers et y ajouter l'utilisateur webdav.

4. Partage WebDAV

Menu : Sharing > WebDAV Shares > Add

  • Nom : WebShare

  • Path : /mnt/Pool1/winshare

  • Cocher Read Only.

  • Décocher "Change User & Group Ownership".

5. Service WebDAV

Menu : Services > WebDAV (éditer)

  • Protocole : HTTPS

  • Port : 8081

  • Certificat : TrueNas_certificate

  • Authentification : Digest

  • Créer un mot de passe WebDAV.

  • Démarrer le service.

6. Accès depuis le navigateur

Taper l'URL complète avec le nom du partage : https://172.16.20.200:8081/WebShare Accepter l'avertissement de sécurité (dû au certificat auto-signé) et s'identifier avec l'utilisateur webdav.
