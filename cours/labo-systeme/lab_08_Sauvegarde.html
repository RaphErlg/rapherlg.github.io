PHASE 2 — Création des VM

Pour chaque VM : VirtualBox → Nouvelle → cocher Skip Unattended Installation.

Avant de démarrer chaque VM : Paramètres → Réseau → Adaptateur 1 → NAT Network → LaboBackup.

VM1 — Windows 11 Tiny11 Pro

  • RAM : 4096 Mo, CPU : 2, Disque : 80 Go minimum (Veeam installe des composants sur C: peu importe le chemin choisi)

  • Pendant l'install Windows, à l'écran partitionnement : une seule partition C:, supprimer toutes les partitions existantes avant de cliquer Nouveau

  • Pas de compte Microsoft : choisir "Compte hors connexion" ou "Utilisation limitée"

  • Compte : admin / tttttt

VM2 — Debian 11 XFCE

  • RAM : 2048 Mo, CPU : 2, Disque : 15 Go

  • Lancer Graphical Install

  • Langue : Français, Pays : Belgique, Clavier : Belge

  • Hostname : debian-gui, Domaine : vide

  • Root : tttttt, User : user / tttttt

  • Partitionnement : assisté, disque entier, tout dans une partition

  • Miroir : deb.debian.org

  • Logiciels à installer : Xfce + Utilitaires usuels + Serveur SSH — tout le reste décoché

  • GRUB : Oui, sur /dev/sda

VM3 — Debian 11 texte

  • RAM : 1024 Mo, CPU : 1, Disque : 10 Go

  • Lancer Install (pas Graphical)

  • Mêmes choix que VM2 sauf Hostname : debian-txt

  • Logiciels à installer : Utilitaires usuels + Serveur SSH — aucun environnement graphique

PHASE 3 — Préparation Debian (CRITIQUE — faire avant tout)

⚠️ Si cette étape est oubliée, le déploiement des agents Veeam sera bloqué indéfiniment. APT cherche les paquets sur le CD d'installation qui n'existe plus.

Sur VM2 ET VM3, en root :

su -
nano /etc/apt/sources.list

Mettre un # devant la ligne deb cdrom: :

# deb cdrom:[Debian GNU/Linux 11 ...]/ bullseye main
deb http://deb.debian.org/debian bullseye main
deb http://security.debian.org bullseye-security main

Ctrl+O → Entrée pour sauvegarder, Ctrl+X pour quitter.

Vérifier que APT fonctionne (doit terminer en moins d'une minute) :

apt update

Vérifier que SSH tourne :

systemctl status ssh

Doit afficher active (running). Si ce n'est pas le cas :

systemctl start ssh
systemctl enable ssh

PHASE 4 — Vérification réseau

Récupérer les IP

Sur VM1 Windows (PowerShell) :

Sur VM2 et VM3 (terminal) :

ip a

Dans notre manip : VM1=10.0.5.1, VM2=10.0.5.3, VM3=10.0.5.6.

Désactiver le pare-feu Windows

PowerShell en administrateur :

Set-NetFirewallProfile -All -Enabled False

Alternative propre — créer une règle ICMP au lieu de tout désactiver :

New-NetFirewallRule `
  -DisplayName "Autoriser ICMP ping - Labo" `
  -Direction Inbound `
  -Protocol ICMPv4 `
  -IcmpType 8 `
  -RemoteAddress 10.0.5.0/24 `
  -Action Allow `
  -Profile Any

PHASE 5 — Configuration SSH par clé

Windows génère une paire clé privée (id_rsa) + clé publique (id_rsa.pub). La clé publique est déposée sur les Debian. Veeam utilise ensuite la clé privée pour se connecter automatiquement sans mot de passe.

Générer la paire de clés — sur VM1 Windows (PowerShell)

ssh-keygen -t rsa -b 4096

Aux 3 questions : appuyer Entrée (chemin par défaut, pas de passphrase).

Crée deux fichiers dans C:\Users\admin\.ssh\ :

  • id_rsa → clé privée (ne jamais partager)

  • id_rsa.pub → clé publique (à copier sur les Debian)

Copier la clé vers VM2 — debian-gui (10.0.5.3)

Depuis PowerShell Windows. Au premier ssh, taper yes pour accepter le fingerprint. Mot de passe : tttttt.

ssh user@10.0.5.3 "mkdir -p .ssh"
scp C:\Users\admin\.ssh\id_rsa.pub user@10.0.5.3:.ssh/
ssh user@10.0.5.3 "cat .ssh/id_rsa.pub >> .ssh/authorized_keys"

Se connecter puis passer root pour autoriser la clé sur le compte root :

ssh user@10.0.5.3

Sur le Debian :

su -
mkdir -p /root/.ssh
cp /home/user/.ssh/authorized_keys /root/.ssh/.
exit
exit

Copier la clé vers VM3 — debian-txt (10.0.5.6)

ssh user@10.0.5.6 "mkdir -p .ssh"
scp C:\Users\admin\.ssh\id_rsa.pub user@10.0.5.6:.ssh/
ssh user@10.0.5.6 "cat .ssh/id_rsa.pub >> .ssh/authorized_keys"

ssh user@10.0.5.6

su -
mkdir -p /root/.ssh
cp /home/user/.ssh/authorized_keys /root/.ssh/.
exit
exit

Tester les connexions sans mot de passe

ssh root@10.0.5.3
ssh root@10.0.5.6

Les deux doivent se connecter sans demander de mot de passe.

PHASE 7 — Installation de Veeam

Télécharger le bon produit

URL : https://www.veeam.com/products/veeam-data-platform/backup-recovery.html

Choisir Veeam Backup & Replication Community Edition (gratuit jusqu'à 10 machines). Pas "Veeam Agent for Windows" qui est un autre produit. Taille : ~19,6 Go. Créer un compte Veeam gratuit.

Installer

  1. Lancer l'exécutable extrait

  2. Choisir Veeam Backup & Replication (pas Console seule)

  3. License Agreement : accepter les deux

  4. License key : laisser vide → mode Community Edition automatique

  5. Program Features : laisser par défaut

  6. System Configuration Check : cliquer Install si des prérequis manquent (SQL Server Express, .NET...)

  7. Data Locations : laisser les chemins par défaut

  8. Service Account : LOCAL SYSTEM account

  9. Install — attendre 10-20 minutes

  10. Finish en laissant "Launch console" coché

Première connexion à la console

Au lancement, fenêtre "Specify backup server" avec localhost. Laisser tel quel → Connect.

Si Failed to connect to the backup server localhost : les services n'ont pas fini de démarrer. Attendre 2-3 minutes et réessayer. Si ça persiste, voir le bloc dépannage.

🔧 Dépannage

"Failed to connect to backup server localhost"

# Voir l'état de tous les services Veeam
Get-Service | Where-Object {$_.Name -like "*Veeam*"} | Select-Object Name, Status

# Démarrer les services critiques
Start-Service VeeamCatalogSvc
Start-Service VeeamBrokerSvc
Start-Service VeeamBackupRESTSvc
Start-Service VeeamWebSvc

# Les passer en démarrage automatique
Set-Service VeeamBrokerSvc -StartupType Automatic
Set-Service VeeamCatalogSvc -StartupType Automatic
Set-Service VeeamBackupRESTSvc -StartupType Automatic
Set-Service VeeamWebSvc -StartupType Automatic

Rescan bloqué 20+ minutes

Cause : ligne cdrom non commentée dans /etc/apt/sources.list.

# Vérifier les connexions bloquées
ss -tn
# Si CLOSE_WAIT vers port 80 → APT bloqué

# Tuer les processus APT bloqués
kill -9 $(pgrep -f "apt")
kill -9 $(pgrep -f "http")

# Vérifier que c'est résolu
ss -tn | grep CLOSE

Commenter la ligne cdrom (cf Phase 3), puis relancer le rescan : Inventory → clic droit sur groupe1 → Rescan. Doit se terminer en moins de 30 secondes.

"Out of disk space" pendant l'installation Veeam

Veeam installe des composants sur C: indépendamment du chemin choisi. Recréer la VM Windows avec 80 Go minimum, partition C: unique.

SSH demande toujours un mot de passe

Vérifier que authorized_keys est bien en place sur le Debian :

cat /home/user/.ssh/authorized_keys
cat /root/.ssh/authorized_keys
# Doit afficher une longue ligne commençant par ssh-rsa

Corriger les permissions si nécessaire (SSH est strict là-dessus) :

chmod 700 /home/user/.ssh
chmod 600 /home/user/.ssh/authorized_keys
chmod 700 /root/.ssh
chmod 600 /root/.ssh/authorized_keys
