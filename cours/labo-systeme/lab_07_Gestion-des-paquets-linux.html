1. Création des machines virtuelles (VirtualBox)

Paramètres communs

  • Type : Linux 64-bit

  • Réseau : NAT (par défaut dans VirtualBox)

  • Disque : VDI, allocation dynamique

Debian (VM « Debian »)

  • RAM : 2 Go

  • CPU : 2 vCPU

  • Disque : 20 Go

  • ISO : Debian 13.4 DVD amd64

Fedora (VM « Fedora »)

  • RAM : 3–4 Go

  • CPU : 2 vCPU

  • Disque : 25 Go

  • ISO : Fedora Workstation Live 43 (amd64)

openSUSE (VM « openSUSE »)

  • RAM : 3 Go

  • CPU : 2 vCPU

  • Disque : 25 Go

  • ISO : openSUSE Leap 16.0 (amd64)

2. Installation et configuration de Debian

2.1 Installation Debian (Graphical Install)

  1. Démarrer la VM Debian sur l’ISO.

  2. Choix de langue : Français.

  3. Pays : Belgique.

  4. Clavier : Français.

  5. Nom de machine : debian.

  6. Domaine : laisser vide.

  7. Mot de passe root : tttttt.

  8. Utilisateur :

    • Nom complet : Raph

    • Identifiant : raph

    • Mot de passe : tttttt

  9. Partitionnement :

    • Assisté – utiliser un disque entier

    • Tous les fichiers dans une seule partition

    • Terminer le partitionnement et appliquer les changements

  10. Miroir réseau :

  • Accepter l’utilisation d’un miroir Debian

  • Proxy : laisser vide

  1. Sélection des logiciels :

  • Environnement de bureau Debian

  • Xfce (ou autre bureau léger)

  • Serveur SSH

  • Utilitaires usuels du système

  1. Terminer l’installation et redémarrer.

2.2 Post-install Debian : sudo + SSH

  1. Se connecter en raph dans la VM.

  2. Ouvrir un terminal et passer root :

    su -
# mot de passe : tttttt

  3. Installer sudo et SSH :

    apt update
apt install sudo openssh-server

  4. Ajouter raph au groupe sudo :

    usermod -aG sudo raph

  5. Activer SSH :

    systemctl enable --now ssh

  6. Vérifier les groupes de raph :

    id raph

2.3 Redirection de port SSH (NAT) pour Debian

  1. Éteindre la VM Debian.

  2. Dans VirtualBox : Paramètres → Réseau → Adaptateur 1 → NAT → Avancé → Redirection de ports.

  3. Ajouter une règle :

    • Nom : SSH

    • Protocole : TCP

    • IP hôte : 127.0.0.1

    • Port hôte : 2222

    • IP invité : vide

    • Port invité : 22

  4. Redémarrer la VM Debian.

  5. Depuis la machine hôte Fedora :

    ssh -p 2222 raph@127.0.0.1

3. Installation et configuration de Fedora

3.1 Installation Fedora Workstation

  1. Démarrer la VM Fedora sur l’ISO Live.

  2. Choisir « Install Fedora Workstation ».

  3. Langue : Français, fuseau horaire : Europe/Brussels.

  4. Partitionnement : automatique sur le disque virtuel.

  5. Créer l’utilisateur raph :

    • mot de passe : tttttt

    • cocher l’option pour le rendre administrateur (sudo via groupe wheel).

  6. Terminer l’installation et redémarrer.

3.2 Post-install Fedora : sudo + SSH

  1. Se connecter avec raph.

  2. Ouvrir un terminal :

    sudo dnf upgrade -y
sudo dnf install -y openssh-server
sudo systemctl enable --now sshd
id

    (Vérifier que raph est dans le groupe wheel.)

3.3 Redirection de port SSH (NAT) pour Fedora

  1. Éteindre la VM Fedora.

  2. Dans VirtualBox → Paramètres → Réseau → Adaptateur 1 → NAT → Avancé → Redirection de ports.

  3. Ajouter :

    • Nom : SSH

    • Protocole : TCP

    • IP hôte : 127.0.0.1

    • Port hôte : 2223

    • IP invité : vide

    • Port invité : 22

  4. Redémarrer la VM Fedora.

  5. Tester depuis l’hôte :

    ssh -p 2223 raph@127.0.0.1

4. Installation et configuration d’openSUSE

4.1 Installation openSUSE Leap

  1. Démarrer la VM openSUSE sur l’ISO.

  2. Choisir « Installation ».

  3. Langue : Français, clavier : Français, fuseau horaire : Europe/Brussels.

  4. Partitionnement : accepter la proposition automatique.

  5. Créer utilisateur raph (mot de passe tttttt).

  6. Mot de passe root : tttttt (si demandé).

  7. Terminer l’installation et redémarrer.

4.2 Post-install openSUSE : sudo + SSH

  1. Se connecter en raph.

  2. Depuis la session graphique ou le TTY :

    sudo -i

    (Si sudo -i fonctionne, root est accessible via le mot de passe de raph.)

  3. Configurer sudo si nécessaire (groupe wheel) :

    zypper refresh
zypper install sudo openssh
usermod -aG wheel raph
EDITOR=nano visudo

    Dans visudo, vérifier/décommenter :

    %wheel ALL=(ALL) ALL

  4. Activer SSH :

    systemctl enable --now sshd
id raph

4.3 Redirection de port SSH (NAT) pour openSUSE

  1. Éteindre la VM openSUSE.

  2. Dans VirtualBox → Paramètres → Réseau → Adaptateur 1 → NAT → Avancé → Redirection de ports.

  3. Ajouter :

    • Nom : SSH

    • Protocole : TCP

    • IP hôte : 127.0.0.1

    • Port hôte : 2224

    • IP invité : vide

    • Port invité : 22

  4. Redémarrer la VM openSUSE.

  5. Tester depuis l’hôte :

    ssh -p 2224 raph@127.0.0.1

5. Mise à jour des trois systèmes (Exercice 1)

5.1 Debian

sudo apt update
sudo apt upgrade -y

Si erreur liée à cdrom:// :

  1. Éditer les sources APT :

    sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak
sudo nano /etc/apt/sources.list

  2. Commenter la ligne qui commence par deb cdrom: (ajouter # au début).

  3. Relancer :

    sudo apt update
sudo apt upgrade -y

5.2 Fedora

sudo dnf upgrade -y

(Fedora combine mise à jour des métadonnées et mise à niveau des paquets.)

5.3 openSUSE

sudo zypper refresh
sudo zypper update
sudo reboot   # si noyau ou composants système mis à jour

À retenir :

  • update / refresh : met à jour la liste des paquets disponibles.

  • upgrade / update (selon l’outil) : met à jour les paquets installés vers la dernière version disponible.

6. Recherche d’un paquet : htop (Exercice 2)

6.1 Debian

sudo apt search htop

→ repérer la ligne htop dans la liste (version et description).

6.2 Fedora

sudo dnf search htop

→ htop.x86_64 Interactive process viewer.

6.3 openSUSE

sudo zypper search htop

→ voir le paquet htop dans la liste.

Conclusion : sur les trois distributions, htop est disponible dans les dépôts officiels.

7. Installation de Postman (Exercice 3)

7.1 Debian – Postman via Snap

  1. Installer snapd :

    sudo apt install snapd

  2. Installer Postman :

    sudo snap install postman

  3. Lancer Postman dans la session graphique de la VM :

    postman
# ou
snap run postman

  4. Remarque : en SSH sans forwarding graphique, on obtient une erreur du type « Missing X server or $DISPLAY ». Toujours tester les applications graphiques dans une session graphique.

7.2 Fedora – Postman via Flatpak

  1. Vérifier Flatpak :

    flatpak --version

  2. Ajouter Flathub en mode utilisateur :

    flatpak remote-add --user --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo

  3. Installer Postman (user) :

    flatpak install --user flathub com.getpostman.Postman -y

    (Si une tentative système échoue avec « Flatpak system operation Deploy not allowed for user », basculer en --user.)

  4. Lancer Postman dans la session graphique Fedora :

    flatpak run com.getpostman.Postman

7.3 openSUSE – Postman via Flatpak

  1. Installer Flatpak :

    sudo zypper install flatpak

  2. Ajouter Flathub (système) :

    sudo flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo

  3. Installer Postman :

    sudo flatpak install flathub com.getpostman.Postman -y

  4. Lancer (si session graphique disponible) :

    flatpak run com.getpostman.Postman

Remarque : sur une VM sans interface graphique, l’installation est correcte mais l’application graphique ne peut pas être testée sans serveur d’affichage.

9. Résumé rapide des commandes clés

Debian

  • Mise à jour :

    sudo apt update
sudo apt upgrade -y

  • Recherche d’un paquet :

    sudo apt search nom_paquet

  • Installation paquet officiel (ex. VLC) :

    sudo apt install vlc

  • Snap / Postman :

    sudo apt install snapd
sudo snap install postman

Fedora

  • Mise à jour :

    sudo dnf upgrade -y

  • Recherche paquet :

    sudo dnf search nom_paquet

  • Flatpak / Flathub / Postman :

    flatpak --version
flatpak remote-add --user --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo
flatpak install --user flathub com.getpostman.Postman -y
flatpak run com.getpostman.Postman

openSUSE

  • Mise à jour :

    sudo zypper refresh
sudo zypper update

  • Recherche paquet :

    sudo zypper search nom_paquet

  • Flatpak / Flathub / Postman :

    sudo zypper install flatpak
sudo flatpak remote-add --if-not-exists flathub https://flathub.org/repo/flathub.flatpakrepo
sudo flatpak install flathub com.getpostman.Postman -y
