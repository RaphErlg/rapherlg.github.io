2. Commandes utiles labo

  • cat : affiche le contenu d’un fichier.

  • less : affiche un fichier page par page, permet de naviguer.

  • grep / egrep : recherche de texte avec ou sans expressions régulières.

  • tac : comme cat mais à l’envers (dernière ligne en premier).

  • cut : extrait des colonnes ou des caractères précis de chaque ligne.

  • ps : affiche les processus en cours.

  • sort : trie des lignes de texte.

  • paste : colle des fichiers côte à côte (colonnes).

  • uniq : supprime ou compte les lignes consécutives identiques.

Pour voir le manuel d’une commande :

man nom_de_commande

3. Préparation des fichiers de test (Exercice 0)

  1. Créez le fichier f1 :

cat <<EOF > f1
aaa
bbb
aaa
ddd
EOF

  1. Créez le fichier f2 :

cat <<EOF > f2
eee
hhh
eee
fff
EOF

  • cat <<EOF lit ce que vous tapez jusqu’à la ligne EOF.

  • > f1 ou > f2 redirige ce texte dans le fichier correspondant.

  1. Fusionner alternatives.log et auth.log dans ~/concat.txt (adapter le chemin si besoin, souvent dans /var/log) :

  • Dans less, tapez : /PWD=/home/ puis Entrée.

4.3. grep

  1. Lignes contenant iproute dans bootstrap.log :

grep "iproute" /var/log/bootstrap.log

  1. Compter les lignes contenant Setting up dkms :

grep -c "Setting up dkms" /var/log/bootstrap.log

4.4. tac

Afficher ~/.bash_history en ordre inverse :

tac ~/.bash_history

4.5. cut

  1. Extraire la 5ᵉ colonne de /etc/passwd (colonnes séparées par :) :

cut -d: -f5 /etc/passwd

  1. Afficher les 4 premiers caractères de chaque ligne de /etc/fstab :

cut -c1-4 /etc/fstab

4.6. ps

  1. Tous les processus en cours :

ps aux

  1. Processus de l’utilisateur courant :

ps -u "$USER"

4.7. sort

  1. Trier les lignes de /etc/passwd par ordre alphabétique :

sort /etc/passwd

  1. Pour un tri « alpha-numérique » classique, la même commande suffit (le tri standard est déjà alpha-numérique) :

sort /etc/passwd

4.8. paste

  1. Fusionner les lignes de f1 et f2 côte à côte :

paste f1 f2

  1. Séparer les colonnes par des virgules :

paste -d"," f1 f2

4.9. uniq

  1. Filtrer les lignes répétées consécutives de kern.log :

uniq /var/log/kern.log

En pratique, on trie souvent avant pour grouper les identiques :

sort /var/log/kern.log | uniq

  1. Compter le nombre d’occurrences de chaque ligne :

sort /var/log/kern.log | uniq -c

5. Exercices avec expressions régulières

5.1. Motif de type abc*

Rechercher les lignes contenant alias suivi de zéro ou plusieurs l dans ~/.bashrc :

grep -E "alias l*" ~/.bashrc

5.2. Motif de type abc+

Rechercher les lignes contenant alias suivi de un ou plusieurs l :

grep -E "alias l+" ~/.bashrc

5.3. Motif de type ^[abc]

Lignes commençant par #, c ou e dans ~/.bashrc :

grep -E "^[#ce]" ~/.bashrc

5.4. Motif de type [0-9]$

Lignes se terminant par un chiffre dans /etc/bash.bashrc :

grep -E "[0-9]$" /etc/bash.bashrc

6.0. Préparation

6.1. Exercice 1 – Numérotation des lignes

Consigne : afficher /etc/network/interfaces en numérotant les lignes.

6.2. Exercice 2 – Tri et lignes uniques

Consigne : trier le contenu de f1 et afficher les lignes uniques.

Commande à utiliser :

sort f1 | uniq

À enregistrer dans resultat.txt :

echo "sort f1 | uniq" >> resultat.txt

6.3. Exercice 3 – paste + recherche

Consigne : coller f1 et f2, puis rechercher les occurrences de ea dans le résultat.

Commande à utiliser :

paste f1 f2 | grep "ea"

À enregistrer dans resultat.txt :

echo "paste f1 f2 | grep "ea"" >> resultat.txt

6.4. Exercice 4 – Lignes contenant une adresse email

Consigne : dans /bin/ssh-copy-id, afficher toutes les lignes qui contiennent une adresse email.

Commande à utiliser :

egrep "[a-zA-Z0-9._-]+@[a-zA-Z0-9._-]+\.[a-zA-Z]+" /bin/ssh-copy-id

À enregistrer dans resultat.txt :

echo "egrep "[a-zA-Z0-9._-]+@[a-zA-Z0-9._-]+\.[a-zA-Z]+" /bin/ssh-copy-id" >> resultat.txt

Que signifie l’expression régulière ?

  • [a-zA-Z0-9._-]+ : une suite d’au moins un caractère autorisé (lettre, chiffre, point, tiret, underscore).

  • @ : le caractère arobase.

  • [a-zA-Z0-9._-]+ : le nom de domaine (ex. gmail, hotmail).

  • \. : un point littéral (le \ sert à l’échapper).

  • [a-zA-Z]+ : l’extension du domaine (ex. com, fr, be).
