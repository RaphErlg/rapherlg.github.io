Lab_04 cubic


Synthèse — Labo Live CD avec Cubic

Objectif

Créer un Live CD / Live USB personnalisé à partir d’une image ISO existante (Linux Mint ou dérivé d’Ubuntu), qui servira de boîte à outils de dépannage (récupération de données, diagnostic réseau, etc.).

L’idée :

  1. Préparer une VM Linux fonctionnelle.

  2. Installer l’outil Cubic.

  3. Copier une ISO de base dans un dossier de travail.

  4. Ouvrir l’ISO avec Cubic, modifier les paquets.

  5. Générer une nouvelle ISO.

  6. Tester cette ISO dans une autre VM (ou plus tard sur une clé USB).

Préparation de la VM

Configuration générale

  • VM sous Linux Mint (ou Ubuntu-like).

  • Disque virtuel suffisamment grand (disque dynamique OK).

  • Accès internet.

  • Optionnel mais très pratique :

    • SSH activé pour piloter la VM depuis Windows.

    • Dossier partagé VirtualBox pour transférer les fichiers (ISO, etc.).

    • Instantanés (snapshots) à chaque grande étape.

Installation et activation de SSH (dans la VM)

sudo apt update
sudo apt install openssh-server -y
sudo systemctl enable --now ssh
sudo systemctl status ssh

  • status doit afficher active (running).

Connexion en SSH depuis Windows (NAT + redirection de port)

Dans VirtualBox :

  • Mode réseau : NAT.

  • Redirection de port :

    • Port hôte : 2222

    • Port invité : 22

Depuis PowerShell sur Windows :

ssh -p 2222 ton_user@127.0.0.1

Retenir : en NAT, on vise 127.0.0.1 (localhost) avec le port redirigé, pas l’IP interne de la VM.

Dossier partagé VirtualBox (facultatif mais utile)

Installation des composants invités

sudo apt update
sudo apt install virtualbox-guest-utils virtualbox-guest-x11 -y
sudo reboot

Ajouter l’utilisateur au groupe des dossiers partagés

sudo usermod -aG vboxsf $USER
sudo reboot

Après redémarrage :

ls -l /media/

Tu dois voir un dossier du type :

/media/sf_nom_du_partage

C’est le contenu de ton dossier partagé Windows (ex. D:\dossier_partager_VMs).

Étape 1 — Installer Cubic

Dans la VM :

sudo apt-add-repository ppa:cubic-wizard/release
sudo apt update && sudo apt install cubic

Créer un dossier de travail (par exemple dans Documents) :

mkdir -p /home/$USER/Documents/mon-DVD

Instantané conseillé

  • Snapshot 1 : 1 - Cubic installé

Étape 2 — Copier l’ISO de base dans le dossier de travail

  1. Placer l’ISO de Linux Mint (ou autre dérivé Ubuntu) dans le dossier partagé côté Windows.

  2. Dans la VM, copier cette ISO vers le dossier de travail Cubic, et te mettre propriétaire du fichier.

Exemple :

sudo cp /media/sf_dossier_partager_VMs/linuxmint.iso /home/$USER/Documents/mon-DVD/
sudo chown $USER:$USER /home/$USER/Documents/mon-DVD/linuxmint.iso
ls -lh /home/$USER/Documents/mon-DVD/

Résultat attendu : le fichier linuxmint.iso est présent dans Documents/mon-DVD, avec ton utilisateur comme propriétaire.

Étape 3 — Ouvrir l’ISO dans Cubic

  1. Lancer Cubic (menu Mint ou terminal).

  2. Écran “Project Directory” :

    • Choisir : Documents/mon-DVD

  3. Écran “Original ISO” :

    • Choisir : linuxmint.iso

    • Optionnel : personnaliser nom, description, etc.

  4. Lancer l’extraction.

  5. À la fin, Cubic ouvre un terminal avec un prompt du type :

root@cubic:~#

Ce terminal agit dans le futur Live CD, pas sur le système hôte.

Étape 4 — Personnaliser le système dans root@cubic

4.1. Mettre à jour la liste des paquets

apt update

4.2. Supprimer les paquets inutiles

Exemple typique du labo : supprimer LibreOffice (inutile sur un live de dépannage) :

apt remove --purge libreoffice*

Tu peux aussi utiliser :

apt list
apt show nom_du_paquet

pour explorer les paquets installés et décider quoi retirer.

4.3. Installer les outils de ta boîte à outils

Outils demandés dans l’esprit du labo :

  • chntpw : réinitialisation de mots de passe Windows.

  • nmap : diagnostic / scan réseau.

apt install chntpw
apt install nmap

Outils supplémentaires utiles (exemples) :

apt install htop testdisk gparted curl wget -y

  • htop : surveillance des processus.

  • testdisk : récupération de partitions / données.

  • gparted : gestion des partitions.

  • curl / wget : téléchargements, tests réseau.

4.4. Nettoyage

apt autoremove -y
apt clean

  • autoremove : supprime les dépendances devenues inutiles.

  • clean : vide le cache des paquets pour alléger l’ISO.

Instantané conseillé

  • Snapshot 2 : 2 - Outils installés dans Cubic

Étape 5 — Finaliser dans l’interface de Cubic

  1. Cliquer sur Next (après le terminal).

  2. Écran des paquets :

    • En général, laisser tel quel si le gros ménage est déjà fait.

  3. Écran du noyau / boot :

    • Choisir l’image de démarrage/noyau la plus “générique” (souvent suffixée par -generic).

  4. Écran de compression :

    • Choisir un niveau de compression normal (éviter l’extrême “max” si tu veux que ça reste raisonnable en temps).

  5. Lancer la génération de l’ISO.

À la fin, Cubic produit une nouvelle ISO dans le dossier de projet (souvent Documents/mon-DVD).

Instantané conseillé

  • Snapshot 3 : 3 - ISO générée

Étape 6 — Tester l’ISO personnalisée

L’idée : vérifier que :

  • l’image boote correctement ;

  • les outils installés sont présents ;

  • les paquets supprimés ne sont plus là.

6.1. Récupérer l’ISO vers Windows (optionnel mais pratique)

Dans la VM de création :

sudo cp /home/$USER/Documents/mon-DVD/nom_nouvel_iso.iso /media/sf_dossier_partager_VMs/

L’ISO apparaît alors dans ton dossier partagé Windows (ex. D:\dossier_partager_VMs).

6.2. Créer une VM de test dans VirtualBox

  1. Nouvelle VM, type Linux / Ubuntu 64 bits.

  2. RAM : 2 Go (ou plus).

  3. Sans disque dur (pour simuler un vrai live).

  4. Dans “Stockage” :

    • Attacher ton ISO personnalisé au lecteur CD de la VM.

Démarre la VM : elle doit booter sur ton Live CD.

6.3. Vérifier la présence des outils

Dans le terminal de la VM de test :

nmap
chntpw

  • Si nmap affiche son aide quand tu tapes juste nmap, c’est normal : ça prouve que la commande existe.

  • Si chntpw répond, il est bien installé.

  • Tu peux ouvrir le menu applications pour vérifier que LibreOffice n’est plus présent.

Instantané conseillé

  • Snapshot 4 : 4 - ISO testée et fonctionnelle

Mini-fiche mémo (pour refaire vite la manip)

1. Préparer la VM

  • Créer VM Linux Mint.

  • Internet OK.

  • Disque suffisamment grand.

  • (Optionnel) SSH + dossier partagé VirtualBox.

2. Installer Cubic

sudo apt-add-repository ppa:cubic-wizard/release
sudo apt update && sudo apt install cubic
mkdir -p /home/$USER/Documents/mon-DVD

3. Copier l’ISO de base

sudo cp /media/sf_partage/linuxmint.iso /home/$USER/Documents/mon-DVD/
sudo chown $USER:$USER /home/$USER/Documents/mon-DVD/linuxmint.iso

4. Ouvrir et modifier avec Cubic

  • Lancer Cubic.

  • Project Directory : Documents/mon-DVD.

  • Original ISO : linuxmint.iso.

  • Attendre l’extraction (terminal root@cubic).

Dans root@cubic :

apt update
apt remove --purge libreoffice*
apt install chntpw
apt install nmap
apt install htop testdisk gparted curl wget -y
apt autoremove -y
apt clean

5. Générer l’ISO

  • Cliquer Next.

  • Garder les paquets ou en retirer si nécessaire.

  • Choisir un noyau/boot “générique”.

  • Choisir la compression.

  • Générer l’ISO.

6. Tester l’ISO

  • Créer une VM de test qui boote sur la nouvelle ISO.

  • Vérifier que :

    • la VM boote bien ;

    • nmap, chntpw (et autres outils) existent ;

    • les paquets retirés ne sont plus là.

Avec ce document, tu as tout ce qu’il faut pour refaire le labo en autonomie, comprendre ce que tu fais à chaque étape, et l’adapter plus tard (autres outils, autre ISO, autre usage).
