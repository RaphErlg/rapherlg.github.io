enumerations atelier


1. Rappel struct + typedef

Avant les énumérations, on a rappelé le modèle des structures, parce qu’on va faire pareil avec enum.[^1]

typedef struct heure Heure;

struct heure {
    int heure;
    int minutes;
};

Heure reveil = { 6, 30 };

Idée clé :

  • struct heure { ... }; → définit un nouveau type structuré

  • typedef struct heure Heure; → crée un alias pour éviter d’écrire struct heure partout

  • Heure reveil; → on déclare une variable de ce type, comme un int

On retrouvera exactement ce schéma avec enum.

2. Énumérations simples : principe et syntaxe

2.1. Premier exemple : les couleurs

typedef enum couleur Couleur;

enum couleur {
    ROUGE,
    ORANGE,
    JAUNE,
    VERT,
    BLEU,
    VIOLET
};

void main(void) {
    Couleur couleurCiel    = BLEU;
    Couleur couleurHerbe;
    Couleur couleursSoleil[] = { ROUGE, ORANGE, JAUNE };

    couleurHerbe = VERT;

    if (couleurCiel == ROUGE) {
        printf("Panique !");
    }
}

Ce qu’est une énumération :

  • Une liste de constantes nommées regroupées en un type.

  • Derrière, chaque constante est représentée par un entier.[^1]

Par défaut :

ROUGE  = 0
ORANGE = 1
JAUNE  = 2
VERT   = 3
BLEU   = 4
VIOLET = 5

2.2. Utilisation d’un type énuméré

Tu peux :

  • déclarer des variables de ce type :

Couleur couleurCiel = BLEU;

  • faire des comparaisons :

if (couleurCiel == ROUGE) { ... }

  • créer des tableaux :

Couleur couleursSoleil[] = { ROUGE, ORANGE, JAUNE };

  • les afficher comme des entiers avec %d :

printf("%d\n", couleurCiel); // affiche 4 si BLEU
```[^1]

3. Syntaxe formelle d’un enum

Définition générale :[^1]

enum <tag> {
    <ENUM1>,
    <ENUM2>,
    ...
};

Avec alias :

typedef enum <tag> <Alias>;

Déclaration de variables :

<Alias> x;
<Alias> y = <CONST>;

Règles de valeurs :

  • Premier énumérateur sans valeur explicite → 0.

  • Chaque suivant sans valeur → précédent + 1.

  • Tu peux forcer une valeur :

enum couleur {
    ROUGE = 17,
    ORANGE,    // 18
    JAUNE,     // 19
    VERT       // 20
};

  • Si tu forces aussi ORANGE :

enum couleur {
    ROUGE  = 17,
    ORANGE = 20,
    JAUNE,     // 21
    VERT       // 22
};

Important :

  • Rien n’occupe “les trous” (comme 18 ici) : ce sont juste des entiers sans nom.

  • Une variable d’un type enum est techniquement un int, le compilateur n’interdit pas :

Couleur c = 999;  // accepté, mais logique fausse

4. Petits exercices autour de l’énumération Couleur

4.1. Recherche avec une fonction estDansListe

#include <stdbool.h>

bool estDansListe(Couleur couleur, Couleur liste[], int tailleListe) {
    int i = 0;
    while (i < tailleListe && liste[i] != couleur) {
        i++;
    }
    return i < tailleListe;
}

  • On parcourt le tableau jusqu’à :

    • soit avoir trouvé la couleur (liste[i] == couleur)

    • soit avoir atteint la fin (i == tailleListe)[^1]

Utilisation :

Couleur couleurPreferee = ROUGE;
Couleur couleursSoleil[] = { ROUGE, ORANGE, JAUNE };

if (estDansListe(couleurPreferee, couleursSoleil, 3)) {
    printf("Ta couleur preferee est dans les couleurs du soleil !\n");
} else {
    printf("Ta couleur preferee n'est pas dans les couleurs du soleil.\n");
}

4.2. Affichage des valeurs entières

void afficheCouleurs(void) {
    printf("Rouge : %d\n", ROUGE);
    printf("Orange: %d\n", ORANGE);
    printf("Jaune : %d\n", JAUNE);
    printf("Vert  : %d\n", VERT);
    printf("Bleu  : %d\n", BLEU);
    printf("Violet: %d\n", VIOLET);
}

  • Permet de visualiser les entiers associés.

  • Quand tu forces des valeurs dans l’enum, ça te permet de voir l’effet.

4.3. Afficher le nom plutôt que le numéro

Une fois compilé, les noms ROUGE, ORANGE… n’existent plus, il ne reste que les entiers. Si tu veux afficher "rouge" au lieu de 0, tu dois le coder toi-même.

void afficheNomCouleur(Couleur couleur) {
    char nomsCouleurs[6][7] = {
        "rouge", "orange", "jaune", "vert", "bleu", "violet"
    };
    printf("%s\n", nomsCouleurs[couleur]);
}

Idées importantes :

  • nomsCouleurs est un tableau de 6 chaînes, chacune contenant jusqu’à 6 lettres + \0.

  • La variable couleur est un entier entre 0 et 5 → elle sert d’index dans le tableau.

⚠️ Si tu modifies l’enum pour que ROUGE = 17 par exemple, nomsCouleurs[couleur] devient dangereux : nomsCouleurs[17] sort du tableau → comportement indéfini.

5. Exercices “jours & plages horaires”

5.1. Énumération des jours

typedef enum jour Jour;

enum jour {
    DIMANCHE,
    LUNDI,
    MARDI,
    MERCREDI,
    JEUDI,
    VENDREDI,
    SAMEDI
};

Utilisation :

Jour aujourdhui = LUNDI;
Jour joursCours[] = { LUNDI, MARDI, MERCREDI, JEUDI, VENDREDI };

5.2. Struct Heure et struct PlageHoraire

Rappel Heure :

typedef struct heure Heure;
struct heure {
    int heure;
    int minutes;
};

Nouvelle struct :

typedef struct plageHoraire PlageHoraire;

struct plageHoraire {
    Jour   jour;    // enum
    Heure  heure;   // struct
    Couleur couleur; // enum
};

Initialisation :

PlageHoraire plage1 = { LUNDI, { 8, 30 }, BLEU };

Accès aux champs (arborescent) :

  • plage1.jour

  • plage1.heure.heure

  • plage1.heure.minutes

  • plage1.couleur

5.3. Fonction affichePlageHoraire

void affichePlageHoraire(PlageHoraire plage) {
    char nomsJours[][9] = {
        "DIMANCHE", "LUNDI", "MARDI",
        "MERCREDI", "JEUDI", "VENDREDI", "SAMEDI"
    };

    printf("%s %02d:%02d\n",
           nomsJours[plage.jour],
           plage.heure.heure,
           plage.heure.minutes);
}

  • nomsJours[plage.jour] → même technique que pour les couleurs.

  • %02d → affiche 2 chiffres avec zéro devant si besoin (8 → 08).

5.4. Fonction estPlageValide

Critères de validité :

  • pas samedi ni dimanche

  • heure entre 8h00 et 16h59

#include <stdbool.h>

bool estPlageValide(PlageHoraire plage) {
    return plage.jour != SAMEDI
        && plage.jour != DIMANCHE
        && plage.heure.heure >= 8
        && plage.heure.heure <= 16;
}

5.5. Parcourir un grand tableau de plages

On te donne un tableau PlageHoraire cours[] = { ...50 éléments... };.

Boucle typique :

int taille = sizeof(cours) / sizeof(cours[0]);
for (int i = 0; i < taille; i++) {
    if (estPlageValide(cours[i])) {
        affichePlageHoraire(cours[i]);
    }
}

  • On ne s’arrête pas à la première invalide → on teste toutes les entrées.

  • sizeof(cours) / sizeof(cours[0]) → calcule le nombre d’éléments du tableau.

void reinitialiseCouleur(void) {

void changeCouleur(Couleur couleur) {
    switch (couleur) {
        case ROUGE:  printf("\033[1;31m");     break;
        case ORANGE: printf("\033[38;5;214m"); break;
        case JAUNE:  printf("\033[1;33m");     break;

        case BLEU:   printf("\033[1;34m");     break;
        case VIOLET: printf("\033[1;35m");     break;

}

void affichePlageHoraire(PlageHoraire plage) {
    char nomsJours[][9] = {
        "DIMANCHE", "LUNDI", "MARDI",
        "MERCREDI", "JEUDI", "VENDREDI", "SAMEDI"
    };

    changeCouleur(plage.couleur);
    printf("%s %02d:%02d\n",
           nomsJours[plage.jour],
           plage.heure.heure,
           plage.heure.minutes);
    reinitialiseCouleur();
}

6.2. Les trois opérateurs bit à bit

Ils comparent les bits colonne par colonne.

Soit :

configurationMatin = 0011
configurationSoir  = 1010

  1. OR bit à bit | :

0011
1010
----
1011  (configurationMatinOuSoir)

→ bit = 1 si au moins un des deux bits vaut 1. 2. AND bit à bit & :

0011
1010
----
0010  (configurationMatinEtSoir)

→ bit = 1 seulement si les deux bits valent 1. 3. XOR bit à bit ^ :

0011
1010
----
1001  (configurationMatinXORSoir)

→ bit = 1 si les bits sont différents (un seul des deux est 1).

Important :

  • Ne pas confondre les versions logiques (||, &&) qui travaillent sur des booléens, avec les versions bit à bit (|, &, ^) qui travaillent sur les bits d’un entier.

6.3. Bits = salles allumées

On code 4 pièces dans les 4 bits :

Bit (de gauche à droite)

Salle

4ᵉ

Cuisine

3ᵉ

Salle de bains

2ᵉ

Salon

1ᵉʳ

Chambre

Exemple :

  • 0b0101 → SDB et Chambre allumées (bits 3 et 1 à 1)

  • 0b1010 → Cuisine et SDB allumées

6.4. Effet des opérations avec un “masque unitaire”

Si on combine une configuration quelconque avec un bit vector où un seul bit est à 1 (par ex. 0b0010) :

  • config | masque → allume ce bit (met à 1), ne change pas les autres.

  • config & masque → garde uniquement ce bit, met le reste à 0.

  • config ^ masque → inverse ce bit (0→1, 1→0), ne change pas les autres.

Applications typiques :

  • allumer une salle → config = config | SALON;

  • éteindre partout sauf une salle → config = config & SALON;

  • basculer l’état d’une salle → config = config ^ SALON;

6.5. Tester un bit : aSalleAllumee

bool aSalleAllumee(int configuration, int salle) {
    return (configuration & salle) != 0;
}

  • configuration & salle → renvoie un bit vector avec soit 0 partout, soit 1 uniquement sur le bit de la salle si elle est allumée.

  • Les parenthèses sont importantes, car != a une priorité plus forte que &.

7. Énumérations binaires

Problème : écrire des constantes comme 0b0010, 0b0100 devient illisible quand il y a beaucoup de bits.

Solution : utiliser une énumération où chaque constante vaut une puissance de 2.

typedef enum salle Salle;

enum salle {
    CHAMBRE = 0b0001,  // 1
    SALON   = 0b0010,  // 2
    SDB     = 0b0100,  // 4
    CUISINE = 0b1000   // 8
};

Configurations lisibles :

int configurationMatin = SALON | CHAMBRE;   // Salon + chambre
int configurationSoir  = SDB | CUISINE;     // SDB + cuisine
int toutAllume         = SALON | CHAMBRE | SDB | CUISINE;

Pourquoi puissances de 2 ?

  • Chaque valeur correspond à un seul bit.

  • Toute somme/combinaison permet de retrouver exactement quelles constantes sont présentes (pas de collision).[^1]

Exemple :

  • SALON | SDB = 0b0010 | 0b0100 = 0b0110 = 6

  • 6 ne peut venir que de 2 + 4, donc SALON + SDB.

    return (config & s) != 0;
