// Simple

printf("%s", noms[jour]);

// Binaire (bit flags)
typedef enum salle Salle;
enum salle {
    CHAMBRE = 0b0001,
    SALON   = 0b0010,
    SDB     = 0b0100,
    CUISINE = 0b1000
};
int conf = SALON | CHAMBRE;         // combiner
if ((conf & SALON) != 0) {...}      // tester (parenthèses !)
conf = conf & ~SALON;               // éteindre
conf = conf ^ SALON;                // basculer


// ====== POINTEURS ======
int n = 5;
int* p = &n;                        // p contient l'adresse de n
*p = 10;                            // n vaut maintenant 10

void incr(int* p) { (*p)++; }       // modif via pointeur
incr(&n);

// struct : -> au lieu de (*p).
Article* pa = &article;
pa->prix = 1.5;                     // = (*pa).prix

// Tableaux = pointeurs
int tab[] = {1, 2, 3};
int* p = tab;                       // = &tab[0]
p++;          // avance d'1 cellule (pas d'1 octet)
*p            // = tab[1]
tab[i] == *(tab+i)                  // équivalence


// ====== FICHIERS ======
FILE* f = fopen("data.txt", "r");   // "r" lecture, "w" écriture
if (f == NULL) { /* erreur */ }

fprintf(f, "Total : %.2f\n", t);    // écrire
fscanf(f, "%d", &n);                // lire (mot par mot)
fgets(ligne, 100, f);               // lire une ligne (avec espaces)

while (!feof(f)) {...}              // attention au piège (cf. section 13)
while (fgets(l, 100, f) != NULL) {...}  // plus robuste

fclose(f);

// Ligne de commande
void main(int argc, char* args[]) {
    // args[0] = nom du programme
    // args[1] = 1er argument, etc.
}

Règles de Clean Code

  • Une ligne = une instruction. Pas de déclarations multiples sur une ligne.

  • Noms significatifs. i, j OK comme indices ; tout le reste doit dire ce qu'il fait.

  • Évitez les conditions négatives quand possible.

  • Pas de break/return au milieu d'une boucle (sauf switch). Préférez une double condition while.

  • Pas de "rustines" : ajouter des ++ au hasard jusqu'à ce que ça marche. Comprenez le bug avant de corriger.

  • Une fonction = une responsabilité. Si le nom contient "et", c'est probablement à découper.

  • Indentez systématiquement. Tab ou 4 espaces, peu importe, mais cohérent.

  • Commentez le pourquoi, pas le quoi. Le code dit déjà quoi ; le commentaire explique pourquoi.

Constantes

Deux façons de déclarer une constante :

// 1. Avec #define (préprocesseur, pas de type)
#define TAILLE 50
#define PI 3.14159

// 2. Avec const (variable typée non modifiable)
const int TAILLE = 50;
const double PI = 3.14159;

🔑 Préférence dans ce cours : #define pour les constantes "globales" (taille de tableaux, valeurs métier). Pas de = dans #define !

#define PI = 3.14       // ❌ INTERDIT
#define PI 3.14         // ✅

Transtypage (cast)

Forcer une conversion de type :

int   nbFilles = 12, nbEleves = 25;
double pourcentage = (double)nbFilles / nbEleves * 100;
//                   ↑ sans le cast, 12/25 = 0 (division entière)

⚠️ Piège classique : (double)(a + b) / 3 ≠ (double)((a + b) / 3). Le cast prend juste l'élément suivant. Mettre des parenthèses si besoin.

L-value vs R-value

  • L-value : ce qui peut être à gauche d'un = (une variable, une case de tableau, un membre de struct). C'est une zone mémoire.

  • R-value : ce qui peut être à droite (une valeur, une expression).

x = 5;          // x est l-value, 5 est r-value
5 = x;          // ❌ ERREUR : 5 n'est pas l-value
tab[3] = x;     // tab[3] est l-value

scanf_s — lire

Sous Visual Studio 2022, toujours utiliser scanf_s plutôt que scanf (sinon warning).

scanf_s("%d", &n);           // entier, & OBLIGATOIRE
scanf_s("%lf", &prix);       // double : %lf (pas %f !)
scanf_s(" %c", &c);          // char : espace devant %c pour ignorer le \n du buffer
scanf_s("%s", nom, 50);      // chaîne : pas de &, taille du buffer en 3e argument
scanf_s("%d/%d/%d", &j, &m, &a);   // imposer un format

Gestion du buffer

Quand on lit un int puis un char, le \n (Entrée) traîne dans le buffer et perturbe la lecture suivante. Trois solutions :

// 1. Espace devant %c (recommandé)
scanf_s(" %c", &c);

// 2. getchar() pour manger le \n
scanf_s("%d", &n);
getchar();
scanf_s("%c", &c);

// 3. Vider tout le buffer (fonction utilitaire)
void viderBuffer(void) {
    while (getchar() != '\n');
}

Affectation composée

Raccourcis pour modifier une variable :

5. Contrôle de flux

if / else if / else

if (note >= 16) {
    printf("Distinction");
} else if (note >= 12) {
    printf("Grande satisfaction");
} else if (note >= 10) {
    printf("Reussi");
} else {
    printf("Echec");
}

💡 Pour un seul cas, les accolades sont facultatives. Mais les mettre quand même (lisibilité + évite des bugs au moment d'ajouter une ligne).

switch / case

switch (operateur) {
    case '+':
    case '-':                       // case sans break = on enchaîne
        printf("Addition ou soustraction");
        break;
    case '*':
        printf("Multiplication");
        break;
    case '/':
        printf("Division");
        break;
    default:
        printf("Inconnu");
        break;
}

  • switch ne fonctionne qu'avec des valeurs entières (int, char, enum).

  • Oublier un break = "fall-through" : le code continue dans le case suivant. Parfois utile (comme ci-dessus), mais le plus souvent c'est un bug.

  • default capture tout ce qui n'a pas matché.

for

Quand on connaît le nombre d'itérations :

for (int i = 0; i < n; i++) {
    // i va de 0 à n-1
}

// Décompte
for (int i = 10; i > 0; i--) { ... }

// Pas de 2
for (int i = 0; i < 100; i += 2) { ... }

while

Quand on dépend d'une condition :

while (nb < 0 || nb > 100) {
    printf("Recommencez : ");
    scanf_s("%d", &nb);
}

do / while

Variante : on exécute au moins une fois avant de tester :

int nb;
do {
    printf("Recommencez : ");
    scanf_s("%d", &nb);
} while (nb < 0 || nb > 100);

break, continue, return

  • break : sort de la boucle (ou du switch).

  • continue : passe à l'itération suivante.

  • return : sort de la fonction.

⚠️ Conseil du cours : éviter break et return au milieu d'une boucle. Préférer une double condition dans le while pour sortir proprement. Voir pattern recherche linéaire.

6. Fonctions

Structure

Toujours 3 éléments : prototype, définition, appel.

// 1. PROTOTYPE (avant main, après les typedef)
int maximum(int a, int b, int c);

// 2. APPEL (dans main ou ailleurs)
void main(void) {
    int m = maximum(5, 10, 7);
    printf("%d", m);
}

// 3. DÉFINITION (après main)
int maximum(int a, int b, int c) {
    int max = a;
    if (b > max) max = b;
    if (c > max) max = c;
    return max;
}

💡 Le prototype déclare la signature de la fonction au compilateur, ce qui permet de l'utiliser avant sa définition complète. Sans prototype, le compilateur ne saurait pas vérifier les appels.

Fonctions sans retour (procédures)

void afficheBienvenue(void) {        // void = pas de retour
    printf("Bienvenue !\n");
}

void afficheTableau(int tab[], int taille) {
    for (int i = 0; i < taille; i++) {
        printf("%d ", tab[i]);
    }
}

Paramètres et portée

  • Les paramètres et variables locales ont une portée locale : invisibles ailleurs.

  • À l'appel : paramètres effectifs (les valeurs passées).

  • Dans la déclaration : paramètres formels (les variables internes).

void afficheMax(int n1, int n2, int n3) {   // n1, n2, n3 = formels
    int max;                                 // local
    ...
}

void main(void) {
    int a = 5, b = 10, c = 7;
    afficheMax(a, b, c);                     // a, b, c = effectifs
    afficheMax(17, 9, -3);                   // littéraux OK aussi
}

Passage par valeur (par défaut)

Quand on passe un type simple (int, double, char, bool, struct), la fonction reçoit une copie. Modifier la copie ne change pas l'original.

void modifier(int x) {
    x = 100;                // modifie la COPIE
}

void main(void) {
    int n = 5;
    modifier(n);
    printf("%d", n);        // affiche 5, pas 100
}

Pour qu'une fonction modifie une variable de l'appelant, 2 solutions :

  1. Retourner la valeur modifiée et la ré-affecter : n = modifier(n);

  2. Passer par adresse (= pointeur). Voir section 12.

Cas particulier : les tableaux

Les tableaux sont passés par adresse implicitement. La fonction peut donc modifier le tableau original.

void remplir(int tab[], int n) {
    for (int i = 0; i < n; i++) {
        tab[i] = 0;
    }
}

void main(void) {
    int nombres[5];
    remplir(nombres, 5);    // tableau modifié
}

🔑 Pourquoi cette exception ? En réalité, un tableau est un pointeur vers sa première case (voir section 12). C'est cohérent une fois qu'on a vu les pointeurs.

Retour de structure

Contrairement aux tableaux, on peut retourner une structure complète :

typedef struct date Date;
struct date { int j, m, a; };

Date dateObtenue(void) {
    Date d;
    scanf_s("%d/%d/%d", &d.j, &d.m, &d.a);
    return d;
}

void main(void) {
    Date naissance = dateObtenue();
}

Déclaration et initialisation

// Sans init

// Avec init complète

// Avec taille déduite

⚠️ La taille ne peut pas être une variable. Elle doit être une constante littérale ou un #define. (Q1/Q2)

// Parcours

}

Adresses des cases

Une case en mémoire a une adresse, accessible avec & :

printf("%d\n", scores[i]);       // valeur
printf("%p\n", &scores[i]);      // adresse

Calcul de la taille (utile pour ne pas hardcoder)

int tab[] = {10, 20, 30, 40, 50};
int taille = sizeof(tab) / sizeof(tab[0]);    // = 5

💡 Astuce universelle : sizeof(tab) / sizeof(tab[0]) marche pour n'importe quel type d'élément. C'est la division "taille totale / taille d'un élément".

⚠️ À l'intérieur d'une fonction, sizeof(tab) donne la taille d'un pointeur (8 octets sur 64 bits), pas la taille du tableau ! → Toujours passer la taille en paramètre.

Tableaux et fonctions

// Signature : crochets vides
void rempli(int tab[], int taille) {
    for (int i = 0; i < taille; i++) {
        tab[i] = 0;
    }
}

// Appel : nom du tableau, sans crochets ni indice
int mesNombres[10];
rempli(mesNombres, 10);

🔑 La fonction modifie directement le tableau original (passage par adresse implicite, cf. section 6).

⚠️ Une fonction C ne peut PAS retourner un tableau directement. La technique : passer le tableau de destination en paramètre, et la fonction le remplit.

8. Chaînes de caractères

Une chaîne, c'est un tableau de char terminé par \0

🔑 Le \0 marque la fin de la chaîne. C'est ce qui permet à printf("%s", ...) de savoir où s'arrêter, même si le tableau est plus grand que le contenu utile.

Déclaration

#define T_NOM 50

Saisie

char prenom[50];

// Version 1 : un seul mot
scanf_s("%s", prenom, 50);           // pas de &, taille du buffer en 3e arg

// Version 2 : avec espaces
gets_s(prenom, 50);

9. Tableaux à 2 dimensions

Le concept

Quand on a une grille (échiquier, sudoku, carte, horaire), on utilise un tableau à 2 dimensions. Chaque case est repérée par deux indices : [ligne][colonne].

Déclaration et initialisation

// Sans init
int echiquier[8][8];

// Avec init (lignes encadrées par {})
int horaire[4][13] = {
    {4, 0, 2, 2, 0, 2, 2, 0, 4, 2, 2, 0, 4},
    {0, 0, 4, 0, 4, 0, 4, 0, 4, 0, 0, 4, 0},
    {2, 2, 2, 2, 2, 2, 2, 0, 0, 2, 2, 0, 2},
    {8, 2, 0, 2, 0, 2, 4, 2, 0, 4, 2, 4, 2}
};

// Tout à zéro
int grille[3][4] = {0};

💡 Les accolades par ligne sont purement esthétiques (le compilateur les ignore). Mais elles rendent le code beaucoup plus lisible.

⚠️ Règle d'or : la 2e dimension est OBLIGATOIRE

Déclaration

OK ?

int tab[4][13]

✅

int tab[][13] = {...}

✅ (compilateur déduit le nb de lignes)

int tab[4][]

❌

int tab[][]

❌

Pourquoi ? En mémoire, un tableau 2D est stocké à plat, ligne par ligne. Pour calculer où se trouve tab[i][j], le compilateur fait :

adresse = adresse_base + (i × nb_colonnes + j) × sizeof(element)

→ Il a besoin du nombre de colonnes pour faire ce calcul.

Accès et parcours

tab[ligne][colonne] = 42;
printf("%d", tab[1][6]);                // ligne 1, colonne 6

// Parcours ligne par ligne (le plus naturel)
for (int iL = 0; iL < nbLignes; iL++) {
    for (int iC = 0; iC < nbCols; iC++) {
        printf("%d ", tab[iL][iC]);
    }
    printf("\n");
}

// Parcours colonne par colonne (boucles inversées)
for (int iC = 0; iC < nbCols; iC++) {
    for (int iL = 0; iL < nbLignes; iL++) {
        // ...
    }
}

Tableaux 2D et fonctions

// Signature : la 2e dimension est OBLIGATOIRE
void afficheHoraire(int horaire[][13], int nbCours) {
    for (int i = 0; i < nbCours; i++) {
        for (int j = 0; j < 13; j++) {
            printf("%d ", horaire[i][j]);
        }
        printf("\n");
    }
}

// Appel : juste le nom du tableau
afficheHoraire(monTableau, 4);

Astuce : une ligne = un tableau 1D

// horaire[i] désigne la ligne i, vue comme un tableau 1D
void afficheLigne(int tab1D[], int taille) { ... }

afficheLigne(horaire[2], 13);            // passe la ligne 2 comme un tableau 1D

Cas particulier : tableaux de chaînes

Une chaîne = un tableau de char. Donc un tableau de chaînes = un tableau 2D de char.

Définition

    char   nom[30];

💡 Position dans le fichier : après les #define, avant les prototypes.

11. Énumérations

Le concept

Un enum permet de définir un type dont les valeurs possibles sont une liste finie de constantes nommées. Derrière, ce sont des entiers.

Énumération simple

typedef enum couleur Couleur;
enum couleur {
    ROUGE,     // = 0 par défaut
    ORANGE,    // = 1 (précédent + 1)
    JAUNE,     // = 2
    VERT,      // = 3
    BLEU,      // = 4
    VIOLET     // = 5
};

Couleur c = VERT;
printf("%d\n", c);                       // 4

Forcer des valeurs

enum priorite {
    BASSE  = 1,
    MOYENNE = 5,                          // saute des valeurs
    HAUTE                                 // = 6 (précédent + 1)
};

🔑 L'astuce-clé : utiliser un enum comme index de tableau

Puisqu'un enum est un entier, il peut servir d'index dans un tableau de chaînes :

void afficheNomCouleur(Couleur couleur) {
    char noms[6][7] = { "rouge", "orange", "jaune", "vert", "bleu", "violet" };
    printf("%s", noms[couleur]);
}

⚠️ Si on force des valeurs (comme ROUGE = 17), noms[ROUGE] lit noms[17] → sortie de tableau. À utiliser uniquement avec les valeurs par défaut (0, 1, 2…).

Quand : pour stocker plusieurs valeurs simultanées dans un seul entier (chaque bit = une valeur).

12. Pointeurs

Le problème : le passage par valeur

Quand on passe un type simple (int, double, char, struct) à une fonction, C fait une copie. La modification ne sort pas de la fonction.

void augmenteAge(int age) {
    age++;                  // modifie la COPIE locale
}

void main(void) {
    int monAge = 18;
    augmenteAge(monAge);
    printf("%d", monAge);   // affiche 18, pas 19
}

2 solutions possibles

Solution 1 — Retourner la valeur modifiée

int ageAugmente(int age) { return age + 1; }
monAge = ageAugmente(monAge);

Limite : pour une grosse structure, on copie tout 2 fois (aller-retour) → inefficace.

Solution 2 — Passage par adresse (pointeur)

void augmenteAge(int* pAge) {        // 1. param = adresse
    (*pAge)++;                        // 2. on incrémente la valeur À CETTE ADRESSE
}

augmenteAge(&monAge);                 // 3. à l'appel : &

Syntaxe — les 4 notations essentielles

Syntaxe

Signification

int* ou int *

type "pointeur vers un entier"

&maVariable

adresse de maVariable (opérateur d'adresse)

*pointeur

valeur stockée à l'adresse (déréférencement)

pStruct->champ

champ d'une structure pointée (= (*pStruct).champ)

Convention de nommage

Préfixer les pointeurs par p : pAge, pMentore, pInstruction.

Exemple complet

int n = 42;
int* p = &n;            // p contient l'ADRESSE de n

printf("%d\n", n);      // 42 (valeur de n)
printf("%p\n", &n);     // 0x...  (adresse de n)
printf("%p\n", p);      // 0x...  (même adresse, c'est ce que contient p)
printf("%d\n", *p);     // 42 (valeur À CETTE ADRESSE)

*p = 100;               // modifie n à travers p
printf("%d\n", n);      // 100

Pointeurs et structures : la flèche ->

typedef struct mentore Mentore;
struct mentore {
    char nom[50];
    int  age;
};

void augmenteAgeMentore(Mentore* pMentore) {
    pMentore->age++;                 // raccourci pour (*pMentore).age++
    // pMentore->nom est aussi accessible
}

Mentore moi = { "Alice", 25 };
augmenteAgeMentore(&moi);

🔑 pStruct->champ est strictement équivalent à (*pStruct).champ, mais infiniment plus lisible.

Astuce : passer l'adresse d'un seul champ

augmenteAge(&mentores[i].age);       // adresse du champ age uniquement

Pointeurs et tableaux — la grande équivalence

🔑 En C, un tableau EST un pointeur vers sa première cellule.

int valeurs[] = {18, 19, 20, 21};

// Ces 3 écritures sont STRICTEMENT équivalentes :
&valeurs[0]
valeurs
(int*)valeurs

// Et ces 2 appels font la même chose :
afficheTableau(&valeurs[0], 4);
afficheTableau(valeurs, 4);

3 conséquences importantes

  1. Les tableaux sont automatiquement passés par adresse. Pas besoin de & à l'appel.

  2. Un tableau ne connaît pas sa taille. Il faut TOUJOURS la passer en paramètre.

  3. Dans une signature, int tab[] = int* tab (le compilateur les traite à l'identique).

Arithmétique de pointeur

Les cellules d'un tableau sont contiguës en mémoire. Si un int fait 4 octets :

Cellule

Adresse

tab[0]

1000

tab[1]

1004

tab[2]

1008

tab[3]

1012

Quand on fait p++ sur un int*, C n'avance pas de 1 octet mais de sizeof(int) = 4 octets (= 1 cellule). C s'adapte automatiquement au type pointé.

int valeurs[] = {5, 12, 19, 30, 50};
int* p = valeurs;          // pointe vers valeurs[0]

*p                         // 5
p++;                       // avance d'1 CELLULE (4 octets en mémoire)
*p                         // 12

p += 2;                    // avance de 2 cellules
*p                         // 30

p -= 3;                    // recule de 3 cellules
*p                         // 5

(*p)++;                    // incrémente la VALEUR (valeurs[0] devient 6)
*p                         // 6

Équivalence index ↔ pointeur

tab[i]    ≡    *(tab + i)
&tab[i]   ≡    tab + i

Accéder à tab[3], c'est exactement faire de l'arithmétique de pointeur sous le capot.

Soustraction de pointeurs

Donne une distance en cellules (pas en octets) :

int* p1 = &tab[2];
int* p2 = &tab[7];
int distance = p2 - p1;      // 5 (cellules), pas 20 (octets)

Le piège des priorités (* et ++)

Écriture

Ce qui est incrémenté

*p++

le pointeur (après déréf) → équivaut à *(p++)

*++p

le pointeur (avant déréf) → équivaut à *(++p)

(*p)++

la valeur pointée (après)

++(*p) ou ++*p

la valeur pointée (avant)

💡 Conseil : dans le code qu'on écrit soi-même, toujours mettre des parenthèses, même si elles ne sont pas obligatoires. Ça rend l'intention claire.

Comparer deux pointeurs

p1 == p2          // compare les ADRESSES (les deux pointent au même endroit ?)
*p1 == *p2        // compare les VALEURS (les contenus sont égaux ?)

NULL — l'adresse "rien"

NULL représente une adresse invalide / "pointe vers rien". Très utilisé pour les sentinelles :

FILE* f = fopen("data.txt", "r");
if (f == NULL) {
    printf("Echec d'ouverture");
}

// Sentinelle dans une boucle
while (pInstruction != NULL) {
    execute(pInstruction);
}

13. Fichiers

Les 3 étapes obligatoires

Toute manipulation de fichier suit ce schéma :

  1. Ouvrir avec fopen

  2. Lire ou écrire avec fscanf / fgets / fprintf

  3. Fermer avec fclose

FILE* — le type variable fichier

FILE* fichier;

C'est une variable spéciale qui représente le lien entre ton programme et un fichier sur le disque. Pense-y comme un signet qui pointe vers le fichier.

fopen — ouvrir un fichier

FILE* f = fopen("data.txt", "r");

Mode

Effet

"r"

lecture (read) — le fichier doit exister

"w"

écriture (write) — crée le fichier (ou écrase s'il existe)

⚠️ fopen peut échouer (nom invalide, fichier inexistant en mode "r"…). Dans ce cas il retourne NULL. Toujours vérifier avant de lire/écrire.

FILE* f = fopen("data.txt", "r");
if (f == NULL) {
    printf("Erreur d'ouverture\n");
    return;
}
// ... utilisation
fclose(f);

fprintf — écrire dans un fichier

fprintf(fichier, "Total : %.2f euros\n", total);
fprintf(fichier, "%s a %d ans\n", nom, age);

Identique à printf mais avec le fichier comme premier argument. Tous les formats %d, %s, %lf… fonctionnent à l'identique.

fscanf — lire mot par mot

fscanf(fichier, "%d",  &valeur);
fscanf(fichier, "%lf", &prix);
fscanf(fichier, "%s",  chaine);          // pas de & pour les chaînes

Identique à scanf mais avec le fichier en premier argument. Ignore automatiquement les espaces, tabulations et retours à la ligne pour trouver la prochaine valeur.

fgets — lire une ligne entière

Quand le contenu peut contenir des espaces, fscanf %s s'arrêterait trop tôt. Solution : fgets.

char ligne[100];
fgets(ligne, 100, fichier);

⚠️ fgets inclut le \n final dans la chaîne lue. Pas besoin de re-mettre \n dans le printf derrière.

fclose — fermer

fclose(fichier);

⚠️ Toujours fermer à la fin. Sans fclose, les données écrites peuvent ne pas être sauvegardées sur le disque.

Le curseur de lecture

Quand un fichier est ouvert, un curseur se place au début. Chaque fscanf/fgets avance automatiquement le curseur jusqu'à la prochaine valeur. Pas besoin de gérer sa position.

3\n  7\n  2\n  8\n  1
↑ curseur au départ

(après 2 lectures avec fscanf)

3\n  7\n  2\n  8\n  1
            ↑ curseur après 2 lectures

feof — détecter la fin du fichier

Quand on ne sait pas combien d'éléments contient le fichier :

while (!feof(fichier)) {
    fscanf(fichier, "%d", &valeur);
    printf("%d\n", valeur);
}

⚠️ Le piège de feof

feof ne devient vrai qu'après qu'une lecture a buté contre la fin. Si le fichier se termine par un \n :

3\n
7\n
1\n        ← retour à la ligne ici, puis fin

Quand fscanf lit 1, il s'arrête au \n. Il n'a pas encore buté contre la fin. feof reste faux, on rentre dans la boucle une fois de plus, fscanf ne lit rien (mais la variable garde sa valeur précédente), on affiche 1 une 2e fois.

Solution 1 : la lecture en début + fin de boucle

fscanf(fichier, "%c", &c);                   // 1re lecture AVANT la boucle
while (!feof(fichier)) {
    // traitement de c
    fscanf(fichier, "%c", &c);               // lecture suivante À LA FIN
}

Solution 2 : utiliser le retour de fgets comme condition

while (fgets(ligne, 100, fichier) != NULL) {
    // traitement
}

fgets renvoie NULL quand il n'y a plus rien à lire — pas besoin de feof. Plus robuste, gère le cas du fichier vide automatiquement.

🔑 Règle de pouce :

  • Si tu utilises fgets → préfère sa valeur de retour comme condition.

  • Si tu utilises fscanf → feof est souvent plus naturel (avec le pattern lecture-avant / lecture-après).

Plusieurs fichiers simultanés

On peut avoir plusieurs FILE* ouverts en même temps, chacun avec son propre curseur :

FILE* source      = fopen("source.txt", "r");
FILE* destination = fopen("resultat.html", "w");

char ligne[100];
while (fgets(ligne, 100, source) != NULL) {
    fprintf(destination, "<li>%s</li>\n", ligne);
}

fclose(source);
fclose(destination);

Arguments de la ligne de commande

Pour qu'un programme prenne des arguments quand on le lance depuis la console :

void main(int argc, char* args[]) {
    // argc      : nombre d'arguments (programme inclus)
    // args[0]   : nom du programme
    // args[1]   : 1er argument
    // args[2]   : 2e argument, etc.
}

Lancement : monProg.exe fichier1.txt motArechercher fichier2.txt

  • argc = 4

  • args[0] = "monProg.exe"

  • args[1] = "fichier1.txt"

  • args[2] = "motArechercher"

  • args[3] = "fichier2.txt"

🔑 C'est le système qui remplit argc et args[] automatiquement. Le programme n'a rien à fournir.

Ordre de vérification des erreurs :

void main(int argc, char* args[]) {
    // 1. Vérifier le nb d'arguments AVANT de toucher à args[1]
    if (argc < 2) {
        printf("Usage : %s nom_du_fichier\n", args[0]);
        return;
    }

    // 2. Vérifier que le fichier existe
    FILE* f = fopen(args[1], "r");
    if (f == NULL) {
        printf("Impossible d'ouvrir %s\n", args[1]);
        return;
    }

    // 3. Traitement
    // ...
    fclose(f);
}

Exemple complet : lire un fichier de courses

courses.txt :

Pommes 3.15
Fromage 1.69
Pain 2.99

Programme :

#define _CRT_SECURE_NO_WARNINGS
#include <stdio.h>

void main(void) {
    FILE* f = fopen("courses.txt", "r");
    if (f == NULL) {
        printf("Fichier introuvable\n");
        return;
    }

    char   nom[100];
    double prix;
    double total = 0;

    for (int i = 0; i < 3; i++) {
        fscanf(f, "%s",  nom);
        fscanf(f, "%lf", &prix);
        printf("%s : %.2lf\n", nom, prix);
        total += prix;
    }
    printf("==================\nTotal : %.2lf\n", total);

    fclose(f);
}

Niveau 3 — Annexes

A. Patterns récurrents

Ces "recettes" reviennent dans presque tous les exercices. Les mémoriser permet de gagner énormément de temps.

Pattern 1 — Validation d'entrée

int nbValide(int min, int max) {
    int nb;
    scanf_s("%d", &nb);
    while (nb < min || nb > max) {
        printf("Recommencez : ");
        scanf_s("%d", &nb);
    }
    return nb;
}

Utilisation : int jour = nbValide(1, 31);

Pattern 2 — Recherche linéaire sans break ni return

int indiceArticle(int code, Article catalogue[], int n) {
    int i = 0;
    while (i < n && catalogue[i].code != code) {
        i++;
    }
    return i;       // i == n si pas trouvé
}

🔑 La double condition i < n && pas_trouve est LE pattern à retenir pour toute recherche.

Pattern 3 — Sortie tôt (carré magique)

Pour s'arrêter dès qu'une condition est violée :

void verifieCarreMagique(int carre[][TAILLE]) {
    int sommeRef = sommeLigne(carre, 0);

    for (int i = 1; i < TAILLE; i++) {
        if (sommeLigne(carre, i) != sommeRef) {
            printf("Pas magique\n");
            return;                       // sortie immédiate
        }
    }
    printf("Carre magique avec somme %d\n", sommeRef);
}

Pattern 4 — Tous identiques ?

bool sontTousIdentiques(char tab[], int taille) {
    char ref = tab[0];
    int i = 1;
    while (i < taille && tab[i] == ref) {
        i++;
    }
    return (i == taille);     // true si on est allé jusqu'au bout
}

Pattern 5 — Compter les occurrences

int nbOccurrences(int tab[], int taille, int valeur) {
    int n = 0;
    for (int i = 0; i < taille; i++) {
        if (tab[i] == valeur) n++;
    }
    return n;
}

Pattern 6 — Indice du max

int indiceMax(int tab[], int taille) {
    int iMax = 0;
    for (int i = 1; i < taille; i++) {
        if (tab[i] > tab[iMax]) {
            iMax = i;
        }
    }
    return iMax;
}

Pattern 7 — Recopie en filtrant

void copiePositifs(int source[], int dest[], int taille) {
    int iDest = 0;
    for (int iSrc = 0; iSrc < taille; iSrc++) {
        if (source[iSrc] > 0) {
            dest[iDest++] = source[iSrc];
        }
    }
    while (iDest < taille) {              // remplir le reste de 0
        dest[iDest++] = 0;
    }
}

Pattern 8 — Encodage d'une structure complète

Article articleEncode(void) {
    Article a;
    printf("Nom : ");  gets_s(a.nom, 30);
    printf("Prix : "); scanf_s("%lf", &a.prix);
    return a;
}

Pattern 9 — Garnir un tableau passé en paramètre

// Fonction qui "produit" un tableau de résultats
void recherchePositionsMax(int tab[], int positions[], int taille) {
    // ... remplir positions[]
    // pas besoin de retour, le tableau est modifié directement
}

Pattern 10 — Test ASCII d'un caractère

bool estChiffre(char c) {
    return (c >= '0' && c <= '9');
}
// ou : return isdigit(c);

Pattern 11 — Numéro 1D ↔ coordonnées 2D

// Si on a numéroté les cases d'une grille de taille T ligne par ligne :
int ligne = numero / T;
int col   = numero % T;

Pattern 12 — Lecture d'un fichier inconnu

char ligne[100];
while (fgets(ligne, 100, fichier) != NULL) {
    // traiter ligne
}

D. Glossaire

Adresse mémoire — Localisation d'une variable en RAM. Obtenue avec &.

Alias (de type) — Synonyme créé avec typedef pour raccourcir l'écriture (typedef struct jeu Jeu; → on peut écrire Jeu au lieu de struct jeu).

Argument — Synonyme de paramètre. "Paramètre formel" (dans la définition) vs "paramètre effectif" (à l'appel).

Bit flag — Constante d'un enum binaire, qui représente l'état activé/désactivé d'une caractéristique sur un bit.

Buffer — Zone mémoire temporaire. Le "buffer du clavier" est ce qui contient ce que l'utilisateur a tapé avant que scanf ne le récupère.

Cast — Conversion explicite de type. (double)5 force 5 à être traité comme un double.

Curseur (de fichier) — Position courante dans un fichier ouvert. Avance à chaque lecture/écriture.

Déréférencement — Accéder à la valeur pointée par un pointeur, avec *.

Étiquette (de structure/enum) — Le nom donné à la définition. Dans struct jeu { ... };, jeu est l'étiquette.

fall-through — En switch, quand un case n'a pas de break et que l'exécution continue dans le case suivant.

feof — Fonction qui détecte la fin d'un fichier. Piège : ne devient vrai qu'après qu'une lecture a échoué.

L-value — Élément qui peut être à gauche d'un = (une variable, une case de tableau).

Masque — Valeur binaire utilisée avec & pour isoler plusieurs bits.

Membre (d'une structure) — Un champ d'une struct. jeu.nom : nom est un membre de jeu.

NULL — Pointeur invalide. Représente "pointe vers rien". Utilisé pour signaler une erreur ou une sentinelle de fin.

Passage par adresse — Mode de passage à une fonction où on transmet l'adresse de la variable (via un pointeur). La fonction peut modifier l'original.

Passage par valeur — Mode par défaut en C : la fonction reçoit une copie. Modifier la copie ne change pas l'original.

Pointeur — Variable qui contient une adresse mémoire. Type : int*, double*, Article*, etc.

Précondition — Condition supposée vraie à l'entrée d'une fonction (ex : "le code existe dans le catalogue").

Postcondition — Effet garanti à la sortie d'une fonction (ex : "le tableau positionsMax est garni des indices du max").

Prototype — Déclaration de la signature d'une fonction (avant main), sans son corps. Permet au compilateur de vérifier les appels.

R-value — Élément qui peut être à droite d'un = (toute valeur ou expression).

Sentinelle — Valeur spéciale qui marque une fin ou un cas particulier (ex : -1 pour "non trouvé", NULL pour "plus rien", \0 pour "fin de chaîne").

size_t — Type entier non signé, retour de sizeof. Format %zu.

Transtypage — Synonyme de cast.

typedef — Mot-clé pour créer un alias de type.
